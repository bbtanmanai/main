# /save-session — 세션 저장

지금까지 작업한 내용을 날짜별 파일로 저장해 다음 대화에서 이어받을 수 있도록 합니다.

## 실행 절차

### 1. 컨텍스트 수집
- `git diff HEAD` 로 이번 대화에서 변경된 파일 파악
- 이 대화에서 논의된 결정사항, 시도한 접근법, 발견된 이슈 정리
- 현재 테스트/빌드 상태 파악

### 2. 저장 폴더 확인
```bash
mkdir -p ~/.claude/session-data
```

### 3. 파일 작성
파일명: `~/.claude/session-data/YYYY-MM-DD-<8자 id>-session.md`
오늘 날짜와 짧은 ID 사용. 예: `2026-03-29-ld-abc123-session.md`

### 4. 파일 형식

```markdown
# Session: YYYY-MM-DD

**프로젝트:** LinkDrop V2
**주제:** [이 세션에서 작업한 것 한 줄 요약]
**마지막 업데이트:** [현재 시각]

---

## 무엇을 만들었나

[작업 목표와 맥락 설명. 시스템 내 위치 포함.]

---

## 작동하는 것 (근거 포함)

- **[항목]** — 확인 방법: [테스트 통과 / 브라우저 확인 / API 응답 등]

아직 아무것도 확인되지 않았으면: "아직 확인된 항목 없음"

---

## 작동하지 않은 것 (이유 포함)

- **[시도한 방법]** — 실패 이유: [구체적인 에러 메시지 또는 원인]

실패 없으면: "실패한 접근법 없음"

---

## 아직 시도하지 않은 것

- [유망한 접근법 / 아이디어]

---

## 현재 파일 상태

| 파일 | 상태 | 메모 |
|------|------|------|
| `path/to/file` | ✅ 완료 / 🔄 진행중 / ❌ 깨짐 / 🗒️ 미시작 | [내용] |

---

## 내린 결정

- **[결정]** — 이유: [왜 이 방법을 선택했는가]

---

## 블로커 & 미해결 질문

- [해결 안 된 사항]

---

## 다음 단계 (구체적으로)

[재개 시 가장 먼저 할 일을 구체적으로 기술. 생각 없이 바로 시작할 수 있는 수준.]

---

## 환경 / 설정 메모

[서버 실행 방법, 필요한 환경변수, 특이사항 등. 표준적이면 생략.]
```

### 5. 저장 후 사용자에게 표시

파일 전체 내용을 출력하고 확인 요청:
```
세션이 [실제 파일 경로]에 저장되었습니다.

정확한가요? 수정할 내용이 있으면 알려주세요.
```

---

## 주의사항

- 같은 날 여러 번 저장 시 ID를 달리해 별도 파일로 생성
- "작동하지 않은 것" 섹션이 가장 중요 — 다음 세션이 같은 실수를 반복하지 않도록
- BACKLOG.md의 미완료 항목도 "다음 단계"에 반영
