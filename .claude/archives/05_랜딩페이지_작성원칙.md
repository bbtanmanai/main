## 1. 공통 디자인 스타일 — 라이트·다크 공용

`링크드랍 공통디자인스타일.html` 기준을 준수한다.

### 1-1. 배경 규칙 — 3층 합성, 단색 금지

> DESIGN.md §5 §17 §18 준수. 단색만 쓰면 안 됨.

```
레이어 1: var(--bg-base)           — 테마 CSS 변수 (다크: #010828 / 라이트: #dce8f8)
레이어 2: .lg-bg + .lg-blob-1/2/3  — 3개 blob 그라디언트 합성
레이어 3: var(--glass-white)       — glass 카드 표면
레이어 4: 콤보 강조색               — 포인트 요소만 (CTA, 뱃지, 텍스트 3곳 이내)
```

**섹션 배경 직접 지정 금지**: `background: #5A0F2E`, `background: rgba(90,15,46,0.35)` 등 단색·단색 변형을 `section` 또는 `main`의 직접 배경으로 지정하지 않는다.

**동영상 배경 섹션 예외**: `.ld-cine-section`은 `background: #000000` 고정 (DESIGN.md §17). 이 섹션만 테마 독립적이다.

### 1-2. Glass 컴포넌트 기준 (`--glass-*` 변수)

```css
/* 카드·섹션 컨테이너 — 반드시 CSS 변수 사용, 하드코딩 금지 */
background: var(--glass-white);          /* 테마 자동 전환 */
border: 1.5px solid var(--glass-border); /* 테마 자동 전환 */
box-shadow: var(--shadow-glass);
backdrop-filter: var(--blur-sm);         /* blur(8~12px) */
border-radius: 16px;
```

### 1-3. 폰트 원칙

| 용도 | 폰트 | 크기 하한 |
|------|------|-----------|
| 모든 한글·영문 본문 | Pretendard Variable | 18px (시니어 접근성) |

> **주의**: 한글 텍스트에 Anton 절대 금지 (LD-007)

### 1-4. CTA 버튼 최소 터치 영역

- 일반 버튼: `min-height: 48px`, `min-width: 48px`
- 주요 CTA: `min-height: 56px` ~ `64px`

### 2-1. 콤보 색상 적용 규칙 — 포인트 전용 (DESIGN.md §18)

> **핵심 원칙**: 콤보 색상은 포인트다. 섹션 배경·오버레이·glass 카드 베이스에 쓰지 않는다.

```
콤보 배경색 → border-left accent 바, 소형 뱃지 bg (면 배경 금지)
콤보 강조색 → CTA 버튼 1개 bg, 뱃지 bg, 특정 단어 텍스트 강조 (3곳 이내)
오버레이    → rgba(0,0,0,0.45) 중립 어둠만 허용 (콤보 색 오버레이 금지)
```

**허용 예시 (landing1 ⓗ #89E900)**:
```tsx
// ✅ CTA 버튼 (1개)
style={{ background: "#89E900", color: "#222222" }}

// ✅ 챕터 뱃지
style={{ background: "#89E900", color: "#222222", borderRadius: 999, padding: "5px 16px" }}

// ✅ 포인트 텍스트 (span 한정)
<span style={{ color: "#89E900" }}>수익</span>

// ✅ 카드 좌측 accent 바
style={{ borderLeft: "4px solid #89E900" }}
```

**금지 예시**:
```tsx
// ❌ 섹션 배경에 콤보 배경색
<section style={{ background: "#222222" }}>

// ❌ 오버레이에 콤보 색
style={{ background: "rgba(34,34,34,0.45)" }} // → rgba(0,0,0,0.45) 사용

// ❌ 이미지 캡션 그라디언트에 콤보 색
style={{ background: "linear-gradient(transparent, rgba(34,34,34,0.88))" }}
// → linear-gradient(transparent, rgba(0,0,0,0.72)) 사용

// ❌ 모든 카드 테두리에 일괄 적용
const CARD = { border: `1.5px solid rgba(137,233,0,0.18)` } // 과용
```

> landing10은 배경/강조색이 반전(라이트 배경 + 다크 강조)이므로 텍스트 대비를 별도 검증할 것.

## 3. 페이지 구조 원칙 (landing1 기준)

모든 랜딩페이지는 아래 흐름을 따른다.  
각 단계는 섹션 또는 챕터 단위로 구현하고, 흐름 순서를 바꾸지 않는다.

```
┌─────────────────────────────────────┐
│  커버 섹션                            │
│  • 동영상 배경 (최소 1개)              │
│  • 페이지 목차 (우측 수직 스테퍼)       │
│  • 주 헤딩 + 서브 헤딩                │
│  • 첫 번째 CTA 버튼                  │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  공감 (Pain) 섹션                    │
│  • 독자가 겪는 고통 포인트 3가지        │
│  • 감성 언어, 1인칭 질문 형식           │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  해결책 (Solution) 챕터들             │
│  • 챕터 단위 세분화 (최소 3챕터)        │
│  • 각 챕터에 외부 이미지 최소 1개       │
│  • 어려운 용어 → 용어 해설 즉시 추가    │
│  • LinkDrop 강점 최대 표현            │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  증거 (Proof) 섹션                   │
│  • 실제 사용자 후기 / 수익 데이터       │
│  • 수치 강조 카드                     │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  마무리 CTA 섹션                      │
│  • 강한 행동 유도 문구                 │
│  • 주 CTA 버튼 (콤보 강조색 적용)      │
│  • 마스코트 등장 여부 검토 (여백 시)    │
└─────────────────────────────────────┘
```

## 5. 용어 해설 및 LinkDrop 강점 표현 원칙

### 5-1. 용어 해설 규칙

- 업계 전문 용어, 영어 약어, 기술 용어가 등장하는 즉시 **같은 챕터 내** 용어 해설 추가
- 표현 방식은 아래 중 하나를 선택:

**방식 A — 인라인 툴팁 카드** (카드 형태, 본문 바로 아래)
```tsx
<div style={{ background: "rgba(0,0,0,0.4)", borderLeft: "4px solid {강조색}", padding: "16px 20px", borderRadius: 8, marginTop: 12 }}>
  <p style={{ color: "{강조색}", fontWeight: 700, margin: "0 0 4px" }}>용어 해설</p>
  <p style={{ margin: 0 }}>트로프(Trope): 독자가 기대하는 장르 공식. 예: 회귀·빙의·이세계 등</p>
</div>
```

**방식 B — 용어집 테이블** (챕터 말미, 복수 용어 2개 이상 시)
```tsx
<table>
  <thead><tr><th>용어</th><th>뜻</th><th>LinkDrop 활용</th></tr></thead>
  <tbody>
    <tr><td>HLS</td><td>HTTP Live Streaming — 영상 분할 전송 표준</td><td>Mux 동영상 배경에 적용</td></tr>
  </tbody>
</table>
```

### 5-2. LinkDrop 강점 표현 필수 항목

각 랜딩페이지에서 아래 강점을 **최소 1개 이상** 명시적으로 언급한다:

| 강점 | 표현 키워드 |
|------|------------|
| AI 초안 자동 생성 | "AI가 초안을 만들고 당신이 완성" |
| 파트너 수당 체계 | "링크 하나로 수당 자동 적립" |
| V3 콘텐츠 파이프라인 | "아이디어 → 대본 → 출판까지 4단계 자동화" |
| 비전문가도 시작 가능 | "코딩·글쓰기 경험 없어도 3주면 완성" |
| 시니어 친화 UX | "디지털 경험 없어도 카카오 상담 하나로 시작" |

## 6. 챕터 구조 및 외부 이미지 원칙

### 6-1. 챕터 분리 규칙

- 솔루션 영역을 **최소 3챕터**로 세분화
- 각 챕터는 `id` 속성 부여 → 목차 스테퍼에서 앵커 링크로 연결
- 챕터 번호 표시: 우측 상단 뱃지(`ld-wg-chapter-badge`) 또는 인라인 원형 숫자

```tsx
{/* 챕터 헤더 예시 */}
<div id="gate1" style={{ marginBottom: 40 }}>
  <span style={{ background: "#89E900", color: "#222222", borderRadius: 999, padding: "4px 14px", fontSize: 13, fontWeight: 800 }}>
    Chapter 3
  </span>
  <h2 style={{ fontSize: 28, fontWeight: 800, marginTop: 16 }}>Gate 1 — 시리즈 설정</h2>
</div>
```

### 6-2. 외부 이미지 첨부 규칙

- 각 챕터당 **최소 이미지 1개** (추후 사용자가 적절한 이미지로 교체 가능하도록 Unsplash placeholder 사용)
- 이미지는 **캡션 + 출처** 함께 표시
- 이미지 크기: 모바일 `100%` width, 데스크톱 `min(100%, 720px)`
- 캡션 스타일: 이미지 하단 absolute 오버레이 (어두운 그라디언트 배경)

```tsx
{/* 이미지 블록 패턴 */}
<div style={{ position: "relative", borderRadius: 12, overflow: "hidden", marginBottom: 32 }}>
  <img
    src="https://images.unsplash.com/photo-XXXXXXX?w=1200&q=80"
    alt="챕터 설명 이미지"
    style={{ width: "100%", height: "auto", display: "block" }}
  />
  <div style={{
    position: "absolute", bottom: 0, left: 0, right: 0,
    background: "linear-gradient(transparent, rgba(0,0,0,0.72))",
    color: "#fff", padding: "20px 24px 16px", fontSize: "0.85rem", fontStyle: "italic",
  }}>
    이미지 설명 캡션 — 추후 실제 이미지로 교체 예정
  </div>
</div>
```
