# 102 — 작업 프로토콜 & 프로젝트 원칙

## 5단계 작업 프로토콜 (모든 작업에 필수 적용)

모든 작업은 아래 5단계를 한 세트로 완수해야 한다. 단계를 건너뛰거나 합치지 말 것.

```
1. 설계   — 변경할 내용·이유·방법을 명확히 설명한다. 코드 수정 없음.
2. 승인   — 사용자가 "진행", "OK", "해줘" 등 명시적으로 허가할 때까지 대기.
3. 실행   — 승인된 설계 범위 내에서만 코드/파일을 수정한다.
4. 테스트 — 수정 후 실제 동작을 검증한다 (API 호출, ffprobe, 로그 확인 등).
5. 결과보고 — 테스트 결과를 사용자에게 명확히 보고한다 (성공/실패/수치 포함).
```

- 설계 단계에서 사용자 승인 없이 코드를 수정하지 말 것
- 실행 후 테스트 없이 "완료"라고 보고하지 말 것
- 결과보고는 수치·로그·파일 상태 등 검증 가능한 근거와 함께 제시할 것
- 테스트 실패 시 다시 설계 단계부터 시작할 것

## V3 독립성 원칙

**V3는 V2와 완전히 독립된 프로젝트다.**
사용자가 명시적으로 지정하지 않는 한 `C:\LinkDropV2` 경로의 파일·문서·메모리를 읽거나 참조하지 않는다.

## 설계 원칙

- **Zero-Hands 자동화**: 사용자는 주제만 입력, 결과만 수신
- **저비용 우선**: edge-tts(무료) > Google TTS, Pillow 키프레임 > Opal HTML
- **클립별 재시도**: 실패 클립만 선택 재생성, 전체 재시작 없음
- **개발 백로그**: `.claude/rules/BACKLOG.md` — 모든 미완료 항목 단일 관리
