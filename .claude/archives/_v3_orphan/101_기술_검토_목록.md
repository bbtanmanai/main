- **모델 선정 근거** (2026-04-16 테스트):

  | 모델 | 한국어 | 속도 | 구조 | 결론 |
  |------|--------|------|------|------|
  | Gemma 3 27B (`google/gemma-3-27b-it`) | ✅ 자연스러움 | 12s | JSON 완벽 | **채택** |

  | Llama 3.3 70B | ⚠️ 중국어 혼용 | 15s | 가능 | 부적합 |

  | EXAONE 3.5 32B | ❌ 접근 불가 | - | - | 부적합 |

- **특이사항**: HF 서버리스는 OpenAI SDK 호환 URL이 모델별로 다름 → `_call_huggingface_sync()` 전용 함수로 분리

## T-001. PixiJS — V2 랜딩·캐릭터 소개 페이지

- **상태**: 🔍 검토 중
- **검토 시작**: 2026-04-16
- **배경**: 사용자가 PixiJS 실체를 아직 충분히 파악하지 못한 상태. 도입 가능성 메모 목적으로 기록.
- **논의 요약**:
  - PixiJS는 Flash와 구조적으로 유사 — Canvas 렌더링, DOM 외부, SEO 불가
  - V3 내부 도구(키프레임 프리뷰)에는 CSS로 충분 → **V3 도입 배제 확정**
  - V2 랜딩 히어로·캐릭터 소개 페이지는 하이브리드 구조 조건부 적합 가능
    - 텍스트·CTA는 반드시 DOM 유지 (SEO 보호), Canvas는 비주얼 레이어만
  - PixiJS가 유의미한 경우: 캐릭터 스프라이트 시트(표정·동작 전환), 마우스 인터랙션
  - 그 수준이 아니라면 Lottie(~150KB)가 더 가벼운 대안
- **기술 비교**:

  | 기술 | 번들 | 인터랙티브 | 2D 스프라이트 |
  |------|------|-----------|--------------|
  | CSS + GSAP | ~50KB | 제한적 | 불가 |
  | Lottie | ~150KB | 제한적 | 불가 |
  | PixiJS v8 | ~400KB | 완전 지원 | 완전 지원 |

- **재검토 트리거**: 캐릭터 소개 페이지 UX 수준(스프라이트·인터랙션 필요 여부) 확정 후

## T-002. Claude Code 로컬 LLM 구동 (claude-code-local)

- **상태**: ⏸ 검토 중단
- **검토 시작**: 2026-04-16
- **출처**: [nicedreamzapp/claude-code-local](https://github.com/nicedreamzapp/claude-code-local)
- **개요**: Apple Silicon Mac에서 Claude Code CLI를 로컬 LLM으로 구동하는 Python 프록시 서버. Claude Code(Anthropic API 형식) → 로컬 MLX 모델 API로 변환.
- **기술 구조**:
  ```
  Claude Code CLI → Python 서버(포트 8080) → 로컬 MLX 모델 (Gemma/Llama/Qwen)
  ```
- **모델 스펙**:

  | 모델 | 속도 | RAM |
  |------|------|-----|
  | Gemma 4 31B | ~15 tok/s | 18GB |
  | Llama 3.3 70B | ~7 tok/s | 75GB |
  | Qwen 3.5 122B | ~65 tok/s | 75GB |

- **현재 적용 불가 이유**:
  1. **Apple Silicon 전용** — 현재 환경 Windows 11. 구동 불가.
  2. **모델 품질 격차** — 로컬 모델이 Claude Sonnet 4.6 대비 코드 품질·추론 수준 낮음. V3 복잡 파이프라인에 한계.
  3. **비용 절감 효과 불확실** — Claude API 비용이 현재 병목인지 미확인.
- **재검토 조건**: M4 Mac 도입 시 또는 Claude API 비용이 실질적 부담이 될 때
