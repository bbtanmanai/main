# 07 · /webnovel/* 라우트 전용 디자인 규칙

> 최종 확정: 2026-05-02
> 분리 출처: `01_V2_프론트_설계_확정.md` §13

---

## 1. 개요

`/webnovel/` 하위 모든 라우트는 **링크드랍 공통디자인스타일.html**의 컴포넌트 디자인을 기본으로 적용한다.
V2 전역 Ld* 컴포넌트 디자인 시스템과는 별도로 운영되는 독립 디자인 존(zone)이다.

---

## 2. 공용 디자인 소스

```
C:\LinkDropV2\.claude\archives\링크드랍 공통디자인스타일.html
```

웹소설 라우트에서 UI 컴포넌트를 신규 제작하거나 수정할 때 이 파일을 먼저 참조한다.

---

## 3. 레이어 구조

| 레이어 | 내용 | 비고 |
|--------|------|------|
| 테마 | **라이트 전용** — `WebnovelThemeForcer`로 강제 | 다크/토글 없음 |
| GNB | `WebnovelGnb` 독립 컴포넌트 — `globals.css` 의존 없음 | 배경 `#555555` |
| 레이아웃 | `WebnovelShell` — 좌측 사이드바 182px + 메인 콘텐츠 | 최대 너비 1200px |

---

## 4. 색상 원칙

- **`#FF6A00` (오렌지)**: GNB 메뉴 내비게이션 전용 — 콘텐츠 영역 사용 금지
- **버튼**: `#1E293B` (다크 네이비) 계열
- **활성 상태**: `#111827` + `rgba(15,23,42,0.06~0.10)` 배경
- **보더**: `rgba(15,23,42,0.08~0.14)` 중성 계열
- **배경**: `#F7F3F0` (웜 오프화이트)

---

## 5. 컴포넌트 적용 기준

신규 컴포넌트 제작 시 아래 순서로 참조한다.

1. `링크드랍 공통디자인스타일.html` 에서 해당 컴포넌트 클래스 확인
2. 라이트 테마 변형(`[data-theme="light"]`) CSS 값 추출
3. **inline style** 또는 **CSS Module**로 구현 — `globals.css` 오버라이드 방식 금지
4. 오렌지(`#FF6A00`)는 메뉴에만 — 콘텐츠에선 §4 색상 원칙 적용

## 7. 모바일 반응형

- **GNB**: `md` 미만 → 햄버거 + 오른쪽 슬라이드 드로어
- **사이드바**: `md` 미만 → `hidden` (드로어 내 웹소설 메뉴로 대체)
- **콘텐츠**: 모바일 풀너비

## 9. V2/V3 역할 분리 원칙

### V2 수정 허용 범위

`/webnovel/*` 라우트에서 V2가 직접 수정할 수 있는 범위:

- UI 레이아웃·컴포넌트 구조
- 스타일·색상·애니메이션
- 페이지 라우팅 및 네비게이션
- 로딩/에러/빈 상태 표시

### V2 수정 금지 범위 (V3 전용)

아래 항목은 **V3에서만 수정**한다. V2에서 직접 우회·변경 절대 금지.

- API 엔드포인트 로직 (`/api/v1/public/series` 등)
- 인증·인가 처리 (Bearer 토큰 검증 방식)
- 데이터 모델·DB 스키마
- 비즈니스 로직 (시리즈 생성·수정·상태 전환 등)

### V3에 요청하는 방법

로직 이슈 발견 시 V2에서 직접 고치려 하지 말고, 아래 형식으로 V3 팀에 요청한다.

```
[V3 요청]
- 파일: apps/api/...
- 현상: ...
- 원인: ...
- 요청 사항: ...
```

### 현재 V3 요청 대기 항목

| # | 파일 | 현상 | 요청 사항 |
|---|------|------|---------|
| 1 | `apps/api/routers/public_content.py` | `/api/v1/public/series` 호출 시 401 반환 | `os.getenv("V2_API_SECRET")` → `settings.V2_API_SECRET` (pydantic_settings) 으로 교체 |
