# 디버깅 체크리스트 (원인 추적 우선)

## 진입 순서 — 반드시 이 순서를 따를 것

1. **git diff / git log 먼저 확인** — 최근 변경점에서 원인의 80%가 발견됨
2. **에러 메시지 정확히 읽기** — 경고(warning)도 무시하지 말 것
3. **변경 전 정상 동작 버전 확인** — `git show <commit>:<file>` 로 비교
4. **1가지 가설 → 1가지 테스트** — 여러 개 동시에 바꾸지 말 것

## 접근 방식 선택

문제를 분석한 후 하나의 접근 방식을 선택하고 끝까지 수행할 것.
두 가지 방식 사이에서 고민 중이면 하나를 골라 실행하고, 실패해야만 다른 방식으로 전환.
여러 방식을 동시에 시도하지 말 것.

## 자기 검증

작업 완료 전 반드시 결과를 검증할 것:
- 코드 수정 후 → 실행/테스트로 동작 확인
- 파일 생성 후 → 파일 존재 및 내용 확인 (bat은 CRLF 검증)
- 설정 변경 후 → 적용 결과 확인

## 금지

- 원인 파악 전 코드 수정 시도 (증상 대응 금지)
- 같은 명령어 반복 실행 (brute force 금지)
- 3회 이상 실패 시 반드시 접근 방식 전환
- 우회 시도 전에 근본 원인부터 파악
- 열어보지 않은 코드에 대해 추측 금지

## Windows 특이사항 체크

- **줄 바꿈**: bat/cmd 파일은 반드시 CRLF (`\r\n`)
- **인코딩**: Python은 `-X utf8`, bat은 `chcp 65001`
- **경로**: bat 내부에서 `%~dp0` 사용 (자기 자신 경로 기준)
- **cmd 출력 캡처**: `cmd /c`는 불안정 → `powershell -Command` 우선 사용
