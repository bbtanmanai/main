## 1. 프로젝트 성격

- **V2 역할**: 소비자 접점 앱 (랜딩·결제·회원·파트너·강사·CRM)
- **V3 역할**: 콘텐츠 생산 엔진 (웹소설 자동화 파이프라인) — V2와 완전 분리
- **타겟**: 40~65세 시니어 — 스마트폰은 익숙하나 기술 장벽을 느끼는 비전문가
- **디바이스 우선**: 모바일 → 데스크톱 → 태블릿

## 3. 컴포넌트 트리 (Ld* 전체 목록)

출처: `docs/03-design-system/component-inventory.md`

### A. 기초 컴포넌트 (Primitive)
- `LdButton` — variant: primary/secondary/ghost/danger, size: sm/md/lg/xl
- `LdLinkButton` — 시각적 버튼, 실제 `<a href>` (외부 링크)
- `LdIconButton` — 아이콘 전용, aria-label 필수, 48×48dp
- `LdChip` — variant: neutral/primary/success/warning/danger, 높이 32
- `LdBadge` — 상태 배지 (NEW/HOT/한정/추천/인기) + 숫자 배지
- `LdAvatar` — 원형, 4종 (xs32/sm40/md48/lg64), 한글 성 fallback
- `LdProgressStepper` — 4단계 성장 경로 전용, 수평(데스크톱)/수직(모바일)
- `LdSkeleton` — shimmer 애니메이션, 이미지 blur-up 용도
- `LdToast` — variant: info/success/warning/danger, 5초 자동 닫힘
- `LdBottomSheet` — 모바일 전용, 하단에서 올라옴, 드래그 닫기
- `LdAccordion` — FAQ 전용, height 250ms + opacity 150ms
- `LdTooltip` — 데스크톱 hover 500ms(시니어 대응), 터치 금지

### B. 수익·시뮬레이터 특화 (Earnings)
- `LdEarningsCard` — 3색 강제: direct(파랑)/partner(주황)/class(초록)
- `LdSimulator` — 슬라이더 2개 + 카운트업 800ms
- `LdPricingCard` — 런칭가 강조, 정가→런칭가 카운트 변환 800ms
- `LdStatusStepper` — 4색 고정: #10b981/#6366f1/#eab308/#ef4444 (LD-004)

### C. 랜딩 전용 섹션 컴포넌트
- `LdHeroSection` — 랜딩 §1, fade-up 600ms
- `LdProblemEmpathySection` — 랜딩 §2, stagger 80ms
- `LdV3PipelineSection` — 랜딩 §3, 3단계 순차 fade-in + 화살표 draw 총 1.2s
- `LdGrowthPathSection` — 랜딩 §4, LdProgressStepper 래퍼
- `LdSimulatorSection` — 랜딩 §5, LdSimulator 래퍼
- `LdProofSection` — 랜딩 §6, scroll reveal, 후기 3장
- `LdPricingSection` — 랜딩 §7, LdPricingCard 래퍼
- `LdFAQSection` — 랜딩 §8, 공통 5 + 주제별 2 자동 병합
- `LdFinalCTASection` — 랜딩 §9, 풀블리드 파랑, 외부 카카오톡 링크
- `LdStickyBottomCTA` — 모바일 랜딩 하단 고정 바 (64h)

### D. 홈 전용 컴포넌트
- `LdHomeHero` — 홈 히어로, fade-up 600ms
- `LdTopicCardGrid` — 9개 주제 카드 그리드, stagger 80ms
- `LdTopicCard` — 홈·메가메뉴 공용, hover: shadow + scale 1.02

### E. 네비게이션 컴포넌트
- `LdNavHeader` — 데스크톱/태블릿 (높이 64/72), sticky top-0
- `LdMegaMenu` — 데스크톱 9개 랜딩 3×3 그리드, 200ms fade-in
- `LdMobileAppBar` — 모바일 (높이 56), 햄버거
- `LdMobileNavOverlay` — 풀스크린 슬라이드 다운 250ms, 포커스 트랩
- `LdFooter` — 3컬럼(데스크톱)/스택(모바일), 법적 고지 필수
- `LdSkipLink` — "본문으로 바로 가기", 숨김 → focus만 노출

### F. 레이아웃 컴포넌트
- `LdContainer` — narrow(720)/default(1120)/wide(1280)/bleed(100%)
- `LdSection` — 수직 간격 96~128px, title + action 슬롯
- `LdGrid` — 12컬럼 반응형, gap: 모바일16/태블릿24/데스크톱32
- `LdStack` / `LdCluster` — 세로/가로 간격 래퍼

## 11. 모션 시스템 요약

출처: `docs/03-design-system/motion-guidelines.md`

### 허용 듀레이션
| 용도 | 듀레이션 |
|------|---------|
| 호버 피드백 | 150ms |
| 섹션 reveal | 400~600ms |
| 시네마틱 (파이프라인·스텝퍼) | 600~1200ms |
| 카운트업 (수익·가격 변환) | 800ms |
| 아코디언 height | 250ms |
| 아코디언 내용 opacity | 150ms |

### 핵심 금지
- 자동재생 영상 (음소거 포함)
- 3개 이상 요소 동시 모션
- 3초 이상 지속 루프
- 텍스트 자체 움직임 (컨테이너만 이동, 텍스트 정지)
- 3회/초 이상 깜박임

### prefers-reduced-motion 대응
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

```css
--ld-space-8:  48px;   /* ★ 터치 타깃 최소 */
--ld-space-9:  64px;   /* 섹션 간 */
--ld-space-11: 96px;   /* 랜딩 히어로~첫 섹션 */
--ld-space-12: 128px;  /* 푸터 위 */
```

- 인접 터치 타깃 간 **최소 8px** 간격 (시니어 손가락 14mm 고려)
- 랜딩 히어로·최종 CTA: 64×64dp
