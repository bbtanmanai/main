## 2. 페이스북 자동화 마케팅 (미구현)

### 2-1. 개요

페이스북 페이지에 **템플릿화된 마케팅 게시물**을 올리고,
해당 게시물에 댓글을 단 사용자에게 **대댓글로 링크를 자동 작성**하는 자동화 로직.

### 2-2. 기술 구조

```
[Facebook Graph API]
  ├── 게시물 자동 포스팅     → Pages API  (POST /me/feed)
  ├── 댓글 실시간 감지       → Webhooks   (page/feed 이벤트 구독)
  └── 대댓글 자동 작성       → Comments API (POST /{comment-id}/comments)
```

**실행 흐름:**
1. 스케줄러가 템플릿 게시물을 Graph API로 페이지에 자동 포스팅
2. Webhook 서버가 새 댓글 이벤트를 실시간 수신
3. 조건 분기 후 → 해당 댓글에 링크 포함 대댓글 자동 작성

### 2-3. 구현 방식 선택지

| 방식 | 설명 | 비용 | 유연성 |
|------|------|------|--------|
| Graph API 직접 구현 | Node.js + Webhook 서버 직접 개발 | 서버 비용만 | 최고 |
| ManyChat | 노코드 Facebook 자동화 도구 | 월 $15~+ | 중간 |
| n8n (셀프호스팅) | Webhook + 워크플로우 자동화 | 서버 비용만 | 높음 |

### 2-4. 예상 공수 (Graph API 직접 구현 기준)

| 작업 | 공수 |
|------|------|
| Facebook 개발자 앱 생성 + 페이지 권한 설정 | 0.5일 |
| Webhook 서버 구축 (HTTPS 필수) | 1일 |
| 댓글 감지 → 대댓글 작성 로직 | 1일 |
| 템플릿 포스팅 스케줄러 | 0.5일 |
| 테스트 + 예외처리 | 1일 |
| **합계** | **약 4~5일** |

> ManyChat 활용 시: 설정만으로 1~2일 내 MVP 가능 (단, 월 구독 비용 발생)

### 2-5. 리스크 — Facebook 정책 제약

| 리스크 | 내용 |
|--------|------|
| 스팸 감지 | 댓글마다 링크 대댓글 = 스팸으로 분류될 가능성 높음 |
| 계정/페이지 정지 | 반복 위반 시 페이지 기능 제한 또는 영구 정지 |
| 링크 도달률 저하 | 외부 링크 포함 댓글은 알고리즘이 도달률 자동 하락 처리 |
| Inauthentic Behavior | Facebook 정책상 자동화된 대량 댓글은 명시적 위반 항목 |

### 2-6. 권장 구현 방식 — Messenger DM 전환

> 댓글에 링크 직접 삽입보다 **댓글 → Messenger DM 자동 전환** 방식이
> 정책상 안전하고 전환율도 높다.

```
사용자 댓글 감지
  → 자동 대댓글: "DM으로 자세한 안내 드릴게요!"
  → Messenger 자동 메시지: 링크 + CTA 전송
```

이 구조라면 **ManyChat 무료 플랜으로 3일 내 구축 가능**.

### 2-7. 구현 전 필요 조건

- [ ] Facebook 비즈니스 페이지 개설
- [ ] Facebook 개발자 앱 생성 및 검토 통과
- [ ] Page Access Token (장기 토큰) 발급
- [ ] Webhook 수신 서버 HTTPS 엔드포인트 확보
- [ ] 게시물 템플릿 텍스트·이미지 준비

## 5. 인바운드 자동화 — 핵심 전략 방향

> 2026-05-14 확정. 무차별 스팸형 자동화는 채택하지 않는다.

### 정의

**사용자가 먼저 행동한 후에만 자동화가 작동하는 방식 (Inbound Automation)**

```
[사용자가 먼저 행동]        [시스템이 반응]
댓글에 키워드 입력      →   자동 대댓글 + DM 발송
DM으로 먼저 메시지     →   자동 응답 플로우 시작
스토리 반응 (이모지)   →   DM 자동 발송
링크 클릭 후 메시지    →   맞춤 응답
```

### 스팸형 vs 인바운드 비교

| 항목 | 무차별 스팸형 | 인바운드 자동화 (채택) |
|------|-------------|----------------------|
| 플랫폼 정책 | 위반 위험 | 완전 합법 |
| 전환율 | 낮음 (강요) | 높음 (관심 표명 후) |
| 고객 관계 | 부정적 첫인상 | 긍정적 첫인상 |
| 장기 지속성 | 계정 정지 위험 | 무기한 운영 가능 |

### 플랫폼별 인바운드 트리거

| 플랫폼 | 트리거 | 자동 반응 |
|--------|--------|-----------|
| Instagram | 댓글 키워드 / 스토리 이모지 반응 / DM 첫 메시지 | DM 플로우 시작 |
| Facebook | 댓글 키워드 / Messenger 첫 메시지 | Messenger 플로우 |
| 카카오채널 | 채널 추가 / 키워드 입력 | 자동 응답 |

### V2 랜딩별 적용 예시

```
landing8 (SNS 마케팅)
  게시물: "SNS 자동화 세팅법 궁금하시면 '자동화' 댓글 달아주세요"
  사용자: "자동화" 댓글 작성
  시스템: DM → "안녕하세요! 세팅 가이드 링크입니다" → [landing8 URL]

landing11 (원데이 클래스)
  스토리: "커리큘럼 받고 싶으면 이모지 반응 눌러주세요"
  사용자: 이모지 반응
  시스템: DM → "커리큘럼 PDF + 신청 링크" → [landing11 URL]
```

## 7. 콘텐츠 자동 업로드 — 정책·조건 정리

### 자동 업로드는 문제없다

플랫폼들이 공식 API로 지원하는 기능. 자동 댓글·스팸과 전혀 다른 영역.

| 플랫폼 | 자동 업로드 | 공식 지원 도구 |
|--------|------------|----------------|
| Facebook | ✅ 허용 | Graph API / Buffer / Later |
| Instagram | ✅ 허용 | Instagram Content Publishing API / Buffer |
| YouTube | ✅ 허용 | YouTube Data API |
| TikTok | ✅ 허용 (2023~) | TikTok Content Posting API |
| 네이버 블로그 | ❌ 공식 API 없음 | 수동만 가능 |

> **핵심 구분**: 콘텐츠 자동 업로드 = 허용 / 자동 댓글·팔로우·DM 스팸 = 금지

### 주의 조건 3가지

**1. AI 생성 콘텐츠 표기 의무**

| 플랫폼 | 규정 |
|--------|------|
| YouTube | 실제 사람처럼 보이는 AI 영상·음성 → 라벨 표기 필수 |
| TikTok | AI 생성 콘텐츠 → 라벨 표기 필수 |
| Meta (FB/IG) | 선거·정치 관련은 필수, 일반 콘텐츠는 권장 수준 |

**2. 업로드 빈도 제한 (Rate Limit)**
- 너무 빠른 연속 업로드 = 스팸으로 오인 가능
- 권장: 플랫폼당 하루 1~3개 이내

**3. 알고리즘 도달률 — 정책 문제가 아닌 품질 문제**
- 자동 업로드 자체는 도달률에 영향 없음
- 콘텐츠 품질(체류시간·저장·공유)이 도달률 결정
- 양질의 콘텐츠 전제라면 자동화해도 무관

### V2 최종 자동화 스택

```
콘텐츠 생산 (AI 활용)
  ↓
Buffer / Later        ← 플랫폼별 예약 업로드 (FB·IG·유튜브 동시)
  ↓
자동 업로드 완료
  ↓
ManyChat             ← 댓글 키워드 감지 → DM 인바운드 자동화
  ↓
랜딩 페이지 전환 → 가입 → 구매
```

**결론**: 콘텐츠 생산에만 집중하면 업로드·배포·전환까지 전부 자동화 가능.

## 8. 자동 마케팅 완성 구조 — 핵심 인사이트

> 2026-05-14 대화에서 도출된 최종 결론

### 구조 요약

```
[콘텐츠 레이어]          [배포 레이어]            [전환 레이어]
각 랜딩페이지         →   플랫폼별 자동 포스팅   →   랜딩 → 가입 → 구매
  landing1 (웹소설)         Facebook 게시물
  landing5 (제휴마케팅)  →  댓글 감지
  landing8 (SNS대행)        Messenger DM 발송
  landing11 (원데이클래스)  → 링크 = 해당 랜딩 URL
  ...
```

### 핵심 원리

- **랜딩이 늘어날수록 마케팅 소재도 자동으로 늘어나는 구조**
- 각 랜딩페이지의 콘텐츠(사업 주제·타겟·후킹 포인트)를 그대로 Facebook 게시물 템플릿으로 활용
- 자동화 로직은 한 번만 구축하면 모든 랜딩에 재사용 가능

### 구현 전 결정 사항

| 결정 사항 | 옵션 | 현황 |
|----------|------|------|
| 첫 번째 마케팅 대상 랜딩 | 전환율 높은 1~2개 집중 | 미정 |
| 자동화 도구 선택 | ManyChat (빠름) vs Graph API 직접 구현 (유연) | 미정 |
