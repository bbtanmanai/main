깃허브 가입부터 안티그래비티 연동, 무료 웹사이트 배포까지!

1장. 개념 잡기

Git과 GitHub의 차이

이름이 비슷하지만 커피와 커피숍만큼 완전히 다릅니다

💻 Git (포장 기계)

내 컴퓨터 안에서 돌아가는 깐깐한 '포장 기계'이자 '타임머신'입니다.

인터넷 연결이 없어도 내가 수정한 코드를 기록하고, 과거로 되돌려 줍니다.

☁️ GitHub (인터넷 창고)

포장된 박스를 올려두는 '거대한 온라인 창고'입니다.

구글 드라이브처럼 내 코드를 안전하게 백업하고 전 세계에 배포할 수 있습니다.

한 줄 요약: "내 컴퓨터의 Git(기계)으로 포장해서, 인터넷의 GitHub(창고)로 배송한다!"
2장. 첫걸음

나만의 인터넷 창고 만들기

github.com에 가입하고 코드를 담을 빈 방(Repository)을 만듭니다

1

가입하기

github.com 접속 후 이메일과 닉네임으로 무료 회원가입을 진행합니다.

2

새 창고(Repository) 만들기

로그인 후 우측 상단의 [+] 버튼을 누르고 [New repository]를 클릭합니다.

3

이름 설정 및 생성

Repository name에 프로젝트 이름(예: linkdrop)을 적고, Public(공개) 체크 후 화면 맨 아래 초록색 [Create repository]를 누르면 빈 창고 완성!

3장. 주의사항 🚨

초보자가 가장 많이 하는 치명적 실수!

"용량이 3GB가 넘어서 업로드가 펑 터졌어요! (HTTP 500 에러)"

❌ 에러의 원인

GitHub는 텍스트(코드) 전용입니다. 전체 용량 1GB 초과 또는 100MB 이상 파일이 섞여 있으면 서버가 업로드를 차단합니다.

[범인들]
- 무거운 .zip 압축파일
- 대용량 .pdf / .mp4 영상
- 무거운 부품 폴더 (node_modules)

✅ 완벽한 해결책

무거운 파일은 폴더 밖으로 빼고, 개발 찌꺼기 폴더는 거름망(.gitignore) 파일을 만들어 깃허브에 안 올라가게 걸러냅니다.

[거름망 파일 만들기]
파일명: .gitignore
내용: node_modules/
일반적인 웹사이트 코드 용량은 보통 10~50MB 내외입니다. 무거운 이삿짐은 절대 GitHub에 올리지 마세요!
4장. 터미널 명령어

안티그래비티와 GitHub 연결 (마법의 명령어)

안티그래비티 터미널(PS C:\linkdrop>)에 한 줄씩 치면 됩니다

1. 포장 기계 설치
내 컴퓨터 폴더에 Git을 준비시킵니다.
git init
2. 인터넷 창고 연결
GitHub 창고 주소로 배송길을 뚫습니다.
git remote add origin [주소]
3. 장바구니에 담기
폴더 안의 모든 파일을 스캔합니다. (마침표 필수)
git add .
4. 박스 포장 및 이름표
파일을 묶고 작업 내용을 메모합니다.
git commit -m "첫 업로드"
5. 창고로 쏘아 올리기
GitHub 서버로 최종 배송합니다.
git push -u origin main --force
성공하면 화면에 100% 완료 메시지가 뜨며, GitHub 사이트에 내 파일이 짠! 나타납니다.

계정 A, B, C를 번갈아 로그인하더라도 깃허브(GitHub) 연결 권한은 내 컴퓨터(PC) 자체에 튼튼하게 저장되어 있습니다. 따라서 구글 계정을 바꿔도 아무 문제 없이 동기화 버튼 하나로 코드가 잘 올라갑니다!

1
내 깃허브 창고 화면 상단의 톱니바퀴 모양 [Settings] 탭 클릭

3
