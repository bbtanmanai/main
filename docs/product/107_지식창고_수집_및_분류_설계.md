# 📋 106. 지식창고(Knowledge Vault) 수집 및 분류 설계

**날짜: 2026년 3월 6일**  
**설계자: 공동감독 김감독 (AI)**  
**승인자: 이감독 (사용자)**

---

## 1. 개요 (Overview)
본 문서는 "좋은 재료가 좋은 작품을 만든다"는 이감독님의 철학에 따라, 프로젝트의 근본적인 데이터 수집 및 저장 체계를 재설계합니다. 단순 뉴스 크롤링을 넘어, 고순도 지식을 체계적으로 분류하고 영구 저장하여 필요할 때 즉시 꺼내 쓰는 **'지식 창고(Knowledge Vault)'** 시스템을 구축합니다.

---

## 2. 고순도 원재료 수집 (High-Purity Sourcing)
단순 RSS 뉴스 의존도를 낮추고, 다음의 3대 고순도 소스를 주력으로 삼습니다.

1.  **[유튜브 심층 분석]**: 10만 이상의 전문 지식 채널의 자막(Transcript) 추출.
2.  **[전문 서적/PDF]**: 사용자가 업로드한 전문 자료 및 베스트셀러 요약본.
3.  **[커스텀 인사이트]**: 이감독님이 직접 메모하거나 입력한 고유한 경험 지식.

---

## 3. 창고 분류 체계 (Vault Categorization)
수집된 모든 데이터는 저장 전 AI(Refiner)에 의해 다음 카테고리 중 최소 1개 이상으로 태깅되어 저장됩니다.

- **[H] 건강/웰빙**: 시니어 건강, 운동, 식단, 마음 챙김.
- **[M] 재테크/수익**: AI 부업, 앱테크, 노후 자산 관리, 재취업.
- **[T] 기술/트렌드**: 최신 AI 도구 사용법, 디지털 전환 교육.
- **[W] 지혜/명언**: 동기부여, 인생 철학, 고전 명구.
- **[L] 라이프스타일**: 여행, 취미, 시니어 커뮤니티 정보.

---

## 4. 데이터 적재 규격 (Storage Schema)
모든 데이터는 `packages/data/vault/{category}/{date}_{id}.json` 경로에 표준화된 형식으로 저장됩니다.

```json
{
  "id": "UUID",
  "source_type": "YouTube/PDF/Web",
  "category": "건강/웰빙",
  "tags": ["당뇨", "걷기", "아침루틴"],
  "title": "원문의 핵심 제목",
  "summary": "AI가 3줄로 요약한 핵심 인사이트",
  "raw_content": "전체 원문 데이터 (NotebookLM 참조용)",
  "quality_score": 9.5,
  "status": "GOLD",
  "created_at": "ISO-8601"
}
```

---

## 5. 조립식 생산 로직 (Retrieval-Based Production)
제작 단계(`data_transformer`)는 이제 빈 화면에서 시작하지 않고, 다음 과정을 거칩니다.

1.  **주제 선정**: "오늘은 시니어 당뇨 예방 쇼츠를 만들자!"
2.  **창고 호출**: `vault/건강/` 폴더에서 `tags`에 '당뇨'가 포함된 가장 점수 높은 재료 3개를 추출.
3.  **대본 조립**: 추출된 3개의 재료를 융합하여 **'검증된 지식 기반의 명품 대본'** 완성.

---

## 6. 단계별 실행 계획 (Action Plan)
- [ ] **Step 1**: `packages/data/vault/` 하위에 카테고리별 폴더 생성.
- [ ] **Step 2**: 기존 수집기(`source_fetcher`)를 고순도 소스(유튜브 등) 대응용으로 고도화.
- [ ] **Step 3**: 수집된 데이터를 분류하고 `vault`에 자동 적재하는 `Vault Manager` 로직 구현.
