# 📄 103: 회원 관리 시스템 및 상세 리스트 설계

## 1. 개요 (Overview)
서비스를 이용하는 크리에이터(사용자) 정보를 체계적으로 관리하고, 각 회원의 구독 상태, 콘텐츠 생산 통계, 권한을 제어할 수 있는 관리자용 회원 관리 시스템을 설계한다.

## 2. 주요 기능 사양 (Key Features)

### 2.1. 전체 회원 목록 (User Directory)
- **회원 정보 요약**: 이름, 이메일, 가입일, 최근 접속일 표시.
- **상태 필터링**: 활성, 휴면, 정지 회원 구분.
- **검색 기능**: 이름 또는 이메일 기반 통합 검색.

### 2.2. 구독 및 등급 관리 (Tier Management)
- **등급 표시**: FREE, PRO, BUSINESS 등급별 시각적 구분.
- **수동 등급 조정**: 관리자가 특정 회원의 등급을 즉시 변경하는 기능.

### 2.3. 개인별 생산 통계 (Usage Stats)
- **생산량 추적**: 해당 회원이 생성한 카드뉴스 및 영상 콘텐츠 총합.
- **활동 로그**: 마지막 제작 일시 및 브랜드 설정 현황.

## 3. 기술적 구현 가이드 (Implementation)

### 3.1. 프론트엔드 (Next.js)
- **Path**: `apps/web-new/src/app/(admin)/users/list`
- **UI Component**: 
  - `UserTable`: 데이터 중심의 고밀도 테이블.
  - `TierBadge`: 등급별 색상 배지 (PRO: Blue, Business: Purple).
  - `UserActionMenu`: 회원 정지, 비밀번호 초기화 등 제어 메뉴.

### 3.2. 백엔드 (FastAPI / Supabase)
- **Database**: Supabase `auth.users` 및 `public.profiles` 테이블 연동.
- **Endpoint**:
  - `GET /api/v1/users`: 전체 회원 페이징 목록 조회.
  - `PATCH /api/v1/users/{id}/status`: 회원 상태 변경.

## 4. 성공 기준 (Success Metrics)
- 1,000명 이상의 회원 목록에서도 스크롤 및 필터링이 지연 없이 작동해야 함.
- 회원 등급 변경 시 관련 생산 제한 정책이 실시간으로 적용되어야 함.
