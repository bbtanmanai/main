# NotebookLM 지식창고 데이터 수집 설계안

> 작성일: 2026-03-11
> 연관 시스템: `skill-2-video-longform1` · `content/longform` 페이지
> 목적: 12개 템플릿 전용 NotebookLM 노트북에 체계적으로 소스를 투입하기 위한 기준 정의

---

## 1. 전체 구조

```
사용자 주제 입력
     │
     ▼
템플릿 선택 (12종)
     │
     ├─ Vault 시리즈 (5종) ──► 전용 NotebookLM 노트북 (지식 깊이 중심)
     │                              └─ 도메인 전문 소스 필요
     │
     └─ Content 시리즈 (7종) ──► 전용 NotebookLM 노트북 (방법론 중심)
                                    └─ 제작 전략/실전 가이드 소스 필요
```

NotebookLM 쿼리 품질 = **노트북 소스의 관련도 × 밀도**
소스가 주제와 무관하면 NotebookLM이 "출처 없음" 면책 문구를 붙이고 외부 정보로 대체함.

## 3. 소스 포맷 규격

### 3-1. 지원 타입

| 타입 | 설명 | 장점 | 적합 용도 |
|------|------|------|-----------|
| `text` | 평문 텍스트 직접 입력 | 즉시 투입 가능 | 정제된 요약본, 명언 컬렉션 |
| `url` | 웹 URL | 원문 크롤링 자동 | 뉴스, 블로그, 공식 기사 |
| `file` | PDF / TXT 파일 업로드 | 대용량 가능 | 리포트, 백서, 전자책 |
| `drive` | Google Drive 문서 | 실시간 동기화 | 지속 업데이트 소스 |

### 3-2. 텍스트 소스 작성 기준

```
✅ 좋은 소스
- 핵심 정보만 압축 (500~2000자 권장)
- 한국어 문장으로 작성 (TTS 나레이션 언어와 일치)
- 구체적 사실, 수치, 사례 포함
- 소제목으로 구조화

❌ 나쁜 소스
- 영어 원문 그대로 (NotebookLM이 영어로 답변)
- 1000자 미만 단편 정보 (컨텍스트 부족)
- 광고성 문구, 중복 내용
- 메타 정보 없이 날것 데이터
```

### 3-3. 소스 수량 기준

| 단계 | 소스 수 | 품질 수준 |
|------|---------|-----------|
| 최소 운영 | 5개 이상 | 기본 답변 가능 |
| 권장 | 15~25개 | 다양한 주제 커버 |
| 최적 | 30~50개 | 깊이 있는 시나리오 |

> NotebookLM 무료 계정 한도: **노트북당 소스 최대 50개**, 파일당 최대 500,000자
