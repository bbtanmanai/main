# 사용자 개인화 데이터 저장 및 연동 계획

## 1. 개요
현재 'AI 페르소나 라이브러리'를 포함한 서비스 내에서 발생하는 사용자의 선택(좋아요, 선택된 캐릭터 등) 데이터의 보존 및 시스템 간 연동을 위한 중장기 개발 로드맵을 정의합니다.

## 2. 단계별 저장 전략
### 2.1 1단계: 브라우저 기반 임시 저장 (Short-term)
*   **기술:** Web `LocalStorage`
*   **대상:** 로그인하지 않은 익명 사용자 또는 초기 프로토타입 단계
*   **특징:** 새로고침 시에도 데이터가 유지되나, 브라우저/기기가 변경되면 데이터가 유실됨.
*   **저장 데이터:** `liked_persona_ids` (좋아요 한 캐릭터 목록), `recent_selected_ids` (최근 선택된 캐릭터)

### 2.2 2단계: 클라우드 기반 영구 저장 (Long-term)
*   **기술:** `Supabase DB` (PostgreSQL)
*   **대상:** 로그인한 정식 사용자
*   **특징:** 계정 기반으로 어떤 기기에서든 동일한 페르소나 리스트를 공유함.
*   **테이블 설계(안):**
    *   `user_favorites`: `id`, `user_id`, `character_id`, `created_at`
    *   `user_config`: 유저별 선호 스타일, 기본 설정 값 등

## 3. 서비스 간 데이터 연동 시나리오
저장된 데이터는 다음과 같이 타 서비스 모듈과 유기적으로 연동됩니다.

### 3.1 비디오 제작 파이프라인 (쇼츠/롱폼)
*   **워크플로우:** `[캐릭터 라이브러리]`에서 하트(♥) 선택 -> `[비디오 제작 페이지]` 진입 -> DB에서 `user_favorites` 호출 -> "내 전용 배우" 목록 노출.
*   **이점:** 사용자가 매번 캐릭터를 찾을 필요 없이 즉시 제작 단계로 진입 가능.

### 3.2 캐릭터 일관성(Consistency) 학습
*   **워크플로우:** 유저가 반복적으로 선택하는 캐릭터에 대해 서버 측에서 해당 캐릭터의 그리드 데이터(`faceGrid`, `bodyGrid`)를 우선적으로 캐싱하거나 최적화 학습 진행.

## 4. 보안 및 정책
*   **데이터 무결성:** 외부 JSON 데이터(`content_characterimage.json`)의 `id`가 변경될 경우 DB와의 참조 무결성을 유지하기 위한 매핑 관리 필요.
*   **개인정보 보호:** 사용자별 선택 정보는 마케팅 데이터로 활용될 수 있으나, 개인 식별이 불가능한 형태로 관리함.

## 5. 향후 작업 우선순위
1.  (High) `LocalStorage` 연동을 통한 초기 사용자 경험 개선.
2.  (Medium) Supabase Auth 연동 및 사용자 테이블 생성.
3.  (Low) 사용자 선택 기반의 AI 캐릭터 추천 엔진 도입.
