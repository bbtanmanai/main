# AI 페르소나 라이브러리 (Character Image Library)

## 1. 개요
AI 페르소나 라이브러리는 영상 콘텐츠(쇼츠, 롱폼 등) 제작 시 화자(Narrator) 및 주인공으로 활용될 AI 캐릭터 이미지를 관리하고 선택하는 시스템입니다. 단순한 이미지 갤러리를 넘어, AI 비디오 생성 파이프라인의 핵심 입력 데이터인 '캐릭터 일관성(Character Consistency)' 정보를 패키지 단위로 관리합니다.

*   **URL:** `http://localhost:3000/content/characterimage`
*   **주요 대상:** AI 버추얼 휴먼 및 스토리텔링 기반 콘텐츠 크리에이터
*   **사용자 작업공간:** C:\LinkDropV2\design_inbox\캐릭터
*   **이미지 저장경로:** C:\LinkDropV2\apps\web\public\img\content\character

## 2. 주요 기능
### 2.1 스타일 필터링 (12가지 유형)
사용자의 영상 컨셉에 맞춰 12가지의 다양한 화풍을 즉시 탐색할 수 있습니다.
*   **지원 스타일:** 실사, 클레이, 신카이, 컷아웃라인, Chaotic Ink, Anime, 디지털아트, 벡터, 픽사, 버츄얼3D, 누와르, 그래픽
*   **UI 특성:** Flex-Wrap 방식을 적용하여 가로 스크롤 없이 모든 스타일 칩이 화면에 최적화되어 노출됩니다.

### 2.2 캐릭터 선택 및 관리
*   **멀티 선택:** 한 번에 최대 2개의 캐릭터를 선택할 수 있으며, 하단 플로팅 바를 통해 비디오 제작 단계로 연결됩니다.
*   **♥(좋아요) 기능:** 개별 유저별로 선호 페르소나를 저장하며, 이는 향후 제작 모듈의 기본 배우 리스트가 됩니다.
*   **통합 다운로드:** 각 캐릭터 카드의 다운로드 버튼 클릭 시, 해당 캐릭터의 모든 자산이 포함된 **`.zip` 파일**을 즉시 다운로드합니다.

### 2.3 페이징 처리
*   페이지당 **24개**의 캐릭터를 노출하며, 하단 페이지네이션을 통해 대량의 라이브러리를 쾌적하게 탐색합니다.

## 3. 기술 사양 및 자산 구성 (Asset Structure)
### 3.1 캐릭터별 자산 저장 경로 (ID별 폴더 관리)
이미지는 캐릭터의 고유 ID별로 디렉토리를 생성하여 체계적으로 관리합니다.
*   **기본 경로:** `apps/web/public/img/content/character/[ID]/`
*   **예시:** ID가 `c1`인 경우 `apps/web/public/img/content/character/c1/` 폴더 내에 저장.

### 3.2 캐릭터별 필수 자산 구성 (1세트 규정)
하나의 캐릭터(ID: A)는 반드시 다음의 **3가지 요소가 한 세트**로 해당 ID 폴더 내에 업로드되어야 합니다.

1.  **대표 이미지 (`A.png`):** 라이브러리 목록에 표시되는 고화질 정면 대표 샷.
2.  **그리드 이미지 (Face 또는 Body 중 최소 1개 필수):**
    *   **얼굴 그리드 (`A_face_Grid.png`):** 립싱크 및 표정 생성용 8방향 뷰 데이터.
    *   **바디 그리드 (`A_body_Grid.png`):** 캐릭터 포즈 및 일관성 유지용 4방향 뷰 데이터.
    *   *참고: 영상의 성격에 따라 페이스 또는 바디 중 필요한 그리드를 선택적으로 포함할 수 있습니다.*
3.  **압축 패키지 (`A.zip`):** 위 이미지들을 포함하여 실제 제작 엔진에 투입될 전체 자산 압축 파일.

### 3.3 데이터 구조 (`content_characterimage.json`)
```json
{
  "id": "c1",
  "name": "캐릭터명",
  "style": "realism",
  "tag": "실사/뉴스",
  "image": "/img/content/character/c1/c1.png",
  "faceGrid": "/img/content/character/c1/c1_face_Grid.png", // 선택 사항
  "bodyGrid": "/img/content/character/c1/c1_body_Grid.png", // 선택 사항
  "zipFile": "/img/content/character/c1/c1.zip"             // 필수
}
```

## 4. 제작 가이드 및 규칙
### 4.1 화자(Narrator) 최적화 규칙
1.  **Rule 01. 정면 구도:** 립싱크 왜곡을 방지하기 위해 시선이 정면을 향하는 이미지를 권장합니다.
2.  **Rule 02. 그리드 규격:** `face_Grid`와 `body_Grid`는 AI 인식률 극대화를 위해 반드시 **16:9 가로형** 규격을 준수해야 합니다.
3.  **Rule 03. 명확한 이목구비:** 얼굴에 그늘이 없고 배경과 캐릭터의 경계가 명확해야 비디오 변환 시 퀄리티가 확보됩니다.

## 5. 향후 연동 계획
*   **비디오 파이프라인 연동:** 선택된 캐릭터의 그리드 데이터(`faceGrid` 또는 `bodyGrid`)를 기반으로 즉시 발화 영상 생성.
*   **캐릭터 일관성(Consistency) 모듈:** 저장된 zip 파일 내의 다각도 데이터를 학습하여 다양한 배경에서의 캐릭터 구현.
