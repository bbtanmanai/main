  아이템 4대 카테고리

  1. SETTING (세계관·배경)

  ├── 능력/마법 체계
  │   예) "혈맥을 통해 흐르는 기운, 9단계 각성 구조"

  │   예) "3대 길드가 대립하는 도시 구조"
  └── 규칙·법칙
      예) "사망자는 72시간 후 몬스터로 부활"

  2. CHARACTER (인물·캐릭터)

  ├── 주인공 원형
  │   예) "회귀한 S급 헌터, 전생 기억 보유"
  ├── 조력자 유형
  │   예) "냉혹하지만 의리있는 2인자"
  ├── 빌런 동기
  │   예) "세계를 구하려다 악이 된 천재"
  └── 관계 설정
      예) "적대→동료→연인 3단계 관계 변화"

  3. EVENT (사건·플롯)

  ├── 오프닝 훅
  │   예) "평범한 회사원이 출근 중 게이트에 빨려들어감"
  ├── 반전 트리거
  │   예) "믿었던 동료가 배후 조직의 스파이"
  ├── 클라이맥스 구조
  │   예) "최종 보스와의 대결 중 숨겨진 진실 공개"
  └── 엔딩 유형
      예) "열린 결말: 새로운 게이트 등장 암시"

  4. EMOTION (감정·분위기)

  ├── 감정 촉발 장치
  │   예) "죽은 줄 알았던 가족 재회 장면"
  ├── 긴장감 조성
  │   예) "제한 시간 내 보스 처치 or 도시 소멸"
  ├── 독자 공감 포인트
  │   예) "무시당하던 주인공의 첫 인정 받는 순간"
  └── 문체 감정톤
      예) "건조하고 냉소적, 1인칭 관찰자 시점"

  문제가 되는 부분

  1. 웹소설 장르 문체 재현의 한계

  웹소설 독자가 기대하는 것:
  ├── 짧고 끊기는 문장 리듬
  │   "그랬다. 정말로."
  │   "이건 말이 안 됐다. 근데 사실이었다."
  ├── 1인칭 독백 감정 과잉
  ├── 회차 말미 클리프행어
  │   "그 순간, 문이 열렸다."
  └── 독자와의 직접 소통 문체
      "눈치 챘는가? 이게 복선이었다."

  AI 생성 결과:
  → 문학적으로 정돈된 문장 (웹소설답지 않음)
  → 감정 묘사가 설명적
  → 클리프행어가 약함

  2. 화수별 긴장감 설계 불가

  1화  ████  도입 (AI 잘함)
  5화  ███   전개 (무난)
  10화 ██    중반 위기 (밋밋해짐)
  15화 █     반전 (예측 가능)
  20화 ██    결말 (어설픈 마무리)

  문제: AI는 "재미있는 전개 설계"가 아닌
        "그럴듯한 내용 생성"을 함

  → NLM에 "냉소적" 저장되어 있어도

  def quality_check(chapter_text):
      checks = {
          "문장길이":    avg_sentence_length(chapter_text) <= 20,
          "대화비중":    dialogue_ratio(chapter_text) >= 0.35,
          "클리프행어":  has_cliffhanger(chapter_text[-200:]),
          "설명과잉":    not has_over_explanation(chapter_text),
          "분량":        4500 <= len(chapter_text) <= 5500,
      }

      fail_items = [k for k, v in checks.items() if not v]

      if fail_items:
          # 실패 항목만 재생성 (전체 재생성 없음)
          return regenerate_sections(chapter_text, fail_items)

      return chapter_text
