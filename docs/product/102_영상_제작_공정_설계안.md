## 1. 개요

LinkDrop V2의 핵심 서비스인 **AI 롱폼 영상 자동 제작** 파이프라인 설계서입니다.
사용자는 템플릿과 화풍만 선택하면, NotebookLM 지식 기반 시나리오 → 슬라이드(키프레임) 생성 → Google TTS 더빙 → FFmpeg 합성까지 완전 자동화됩니다.

** 영상 제작 목표 및 지향점 **
- 생성형AI 사용을 최대한 자제를 통한 저비용 대량 이미지 및 비디오 생산이 주목적
- 사용자의 최소 노력으로 70점 이상의 품질의 영상 제작
- 70점 영상의 정의: 스타트 이미지-엔드 이미지를 연결하는 단순한 움직임의 MP4
- 링크드랍이 추구하는 컨텐츠는 블루오션을 지향하는 컨텐츠가 아닌 레드오션(실제 사람들이 좋아하는 많이 본 컨텐츠)전용 컨텐츠입니다.

### 핵심 원칙
- **Zero-Hands 자동화**: 사용자는 주제만 입력, 결과만 수신
- **동적 씬 구성**: 고정 N개 아님 — 시나리오 길이에 따라 3~15개 자동 결정
- **최종 영상 상한**: 65초 이하 (TTS_CHARS_PER_SEC = 4.5자/초 기준)
- **클립별 독립 복구**: 전체 재시작 없이 실패 클립만 재생성
- **Opal 분리**: Opal 렌더링 엔진은 **광고영상 파이프라인(별도)** 에서 사용, 롱폼은 FFmpeg 기반
- **NotebookLM이 이미지 생산**: 시나리오 텍스트 + 화풍 프롬프트 주입 → 텍스트 없는 슬라이드 이미지 직접 생성 → 각 슬라이드 = 씬 키프레임

## 2. UI 3단계 구조

```
[STEP 1] 템플릿 · 화풍
    └─ 지식 템플릿 선택 (12종 NotebookLM 연동)
    └─ 키프레임 화풍 선택 (9종) — 둘 다 선택 시 자동 진행

[STEP 2] 시나리오 · 성우
    └─ 프롬프트 스타일 선택 (6종) → NotebookLM 시나리오 3개 병렬 생성
    └─ 주제 입력 (선택 사항, 미입력 시 템플릿명 사용)
    └─ 시나리오 A/B/C 카드 선택 + 자유 편집
    └─ Google Cloud TTS 성우 선택 (8명 미리듣기)
    └─ → "영상 제작 시작 (FFmpeg)" 버튼

[STEP 3] 영상 제작
    └─ SSE 실시간 파이프라인 진행 표시
    └─ 씬 수 확정 후 클립 슬롯 동적 생성
    └─ 클립별 원형 프로그레스 바
    └─ 최종 URL 다운로드
```

```
    ↳ scenario_gen.py × 3 병렬 실행
         └─ NotebookLM API: 시나리오 텍스트 생성 (directive 주입)
         └─ → {type:'scenario', index, script, preview} × 3
         │
         ▼

         │
         ▼

    ↳ pipeline_server.py 실행
         └─ step0: 인증 확인
         └─ step1: NotebookLM 시나리오 (편집본 있으면 스킵)
                   + art_prompt 주입 → 슬라이드 이미지 생성  ← [미구현]
         └─ step2: 씬 파싱 → 동적 clip_count 확정
         └─ step3: 클립별 TTS MP3 + 키프레임 PNG → FFmpeg  ← [미구현]

         └─ → {type:'done', url}
         │
         ▼
사용자: 최종 영상 다운로드
```

## 4. 백엔드 파이프라인 상세 (Python)

### [0단계] 인증 확인
- `step0_check_auth()` — CDP Bearer Token 유효성 검증
- 세션 파일: `~/.linkdrop-opal/profiles/default/session.json`

### [1단계] NotebookLM 시나리오 + 슬라이드 생성
- `step1_get_script(job)` — NotebookLM Python API 직접 호출
- 템플릿별 전용 노트북 → `APP_NOTEBOOK_MAP` 참조
- 노트북 소스 미투입 시 → `INSIGHT_BANK_ID` fallback
- 6가지 `STYLE_DIRECTIVES` 프롬프트 적용 → 씬 텍스트 생성
- **[미구현]** `art_prompt` 주입 → NotebookLM 슬라이드 이미지 생성 (화풍별 키프레임)
- 출력: `[씬1]~[씬N]` 텍스트 + 슬라이드 이미지 N장

### [2단계] 씬 파싱 및 검증
- `step2_parse_and_validate(job)` — 씬 태그 파싱, TTS 길이 계산
- 동적 클립 수: `len(scenes)` → `job.clip_count` (MIN 3 / MAX 15 / 총합 65초 이하)

### [3단계] 클립 병렬 생성 ❌ 미구현
- 현재: `_mock_render()` 시뮬레이션
- 목표:
  ```
  씬 텍스트
      ├─ Google Cloud TTS → 더빙 MP3
      ├─ NotebookLM 슬라이드 이미지 → 키프레임 PNG
      └─ FFmpeg zoompan Ken Burns → clip_N.mp4
  ```

### [4단계] FFmpeg 서버 병합 ❌ 미구현
- `step4_merge(job)` — 조각 병합 + 트랜지션 + BGM
- 엔드포인트: `api.linkdrop.io` (미배포)

## 5. FFmpeg 합성 레이어 설계

```
clip_01.mp4 ──┐
clip_02.mp4 ──┤  xfade 전환 (zoomin / fade / wipeleft 등, 0.3~0.5초)
clip_03.mp4 ──┤  + 누끼 소품 오버레이 (arrow_up.webm, coin_spin.webm 등)
   ...        ┤
clip_N.mp4 ───┘
    + BGM 믹싱 (-20dB 배경음악)
    + 효과음 포인트 삽입
    → final_YYYYMMDD_HHMMSS.mp4 (65초 이하)
```

| FFmpeg 필터 | 용도 | 상태 |
|------------|------|------|
| `zoompan` | 키프레임 이미지 Ken Burns 줌인/아웃 | ❌ |
| `xfade` | 씬 사이 전환 (zoomin, fade, wipeleft 등) | ❌ |
| `overlay` | 누끼 소품(.webm 투명채널) 오버레이 | ❌ |
| `chromakey` | 그린스크린 누끼 제거 | ❌ |
| `amix` | BGM + 더빙 볼륨 믹싱 | ❌ |

## 6. 설정 파일 구조 (JSON 단일 관리)

모든 프롬프트와 설정은 **`apps/web/src/data/content_longform.json`** 에서 직접 수정합니다.

```
content_longform.json
├── artStyles[]          ← 화풍 9종 + NotebookLM 슬라이드 주입 프롬프트
│   └── notebooklmPrompt ← 이 값이 NotebookLM에 직접 주입됨
├── narrativeStyles[]    ← 시나리오 구성 방식 6종 + directive 프롬프트
│   └── directive        ← 이 값이 NotebookLM 시나리오 생성 프롬프트에 삽입됨
├── templates[]          ← 12종 지식 템플릿 + notebookId
└── voices[]             ← Google TTS 성우 8명 + previewUrl
```

### 프롬프트 수정 방법
```json
// artStyles[i].notebooklmPrompt 수정 예시
{
  "notebooklmPrompt": "각각의 슬라이드가 키프레임이 되도록 해주고 슬라이드에는 텍스트를 넣지 말아줘.\n[원하는 화풍 설명 자유 수정]"
}
```
> ⚠️ `"각각의 슬라이드가 키프레임이 되도록 해주고 슬라이드에는 텍스트를 넣지 말아줘."` 문장은 반드시 포함되어야 합니다.

**3가지 연동 방식 비교**

| 방식 | 설명 | 장점 | 단점 | 검토 의견 |
|------|------|------|------|----------|
| **① Drive 네이티브 소스** | NotebookLM에서 Drive 파일을 직접 소스로 연결 (NLM 공식 기능) | 별도 코드 불필요, Drive 변경 시 NLM 자동 반영 | NLM UI에서 수동 연결 필요 | ⭐ 가장 간단, 즉시 적용 가능 |
| **② Cron 동기화** | 주기적으로 Drive 변경 감지 → NLM API로 소스 추가/삭제 | 완전 자동화 | NLM 소스 관리 API 안정성 불확실 | 🔶 중기 검토 |
| **③ 직접 통합** | Drive 파일을 다운로드 후 텍스트 추출 → NLM API 직접 주입 | 완전 제어 가능 | 개발 복잡도 높음 | 🔷 장기 검토 |

**완성 생태계 흐름 (목표)**

```
[콘텐츠 제작자]
    │ Drive에 소스 파일 업로드
    ↓
[Google Drive - 소스 저장소]
    │ Cron 동기화 (DRIVE_NOTEBOOK_MAP 기준)
    ↓
[NotebookLM - AI 처리 엔진]
    │ 주제 + 화풍 프롬프트 주입
    ↓
[LinkDrop 파이프라인]
    │ 시나리오 생성 → TTS → FFmpeg 합성
    ↓
[최종 영상]
    │ Google Drive 자동 저장 (16-1)
    ↓
[옴니채널 배포] — 유튜브 / 인스타 / 틱톡
```
