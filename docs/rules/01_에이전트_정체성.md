# AI 에이전트 설명서 (LinkDrop V2)

본 문서는 **기초 지식 인프라(Foundation)** 구축을 최우선으로 하며, 그 위에서 작동하는 **오팔(Opal) 영상 생산(Application)** 체계를 정의합니다.

---


  1. 에이전트 (Agent) - "일꾼 / 비즈니스 로직"
   * 정의: 특정 비즈니스 목적을 달성하기 위해 작성된 독립된 프로그램(스크립트)입니다.
   * 위치: apps/api/services/agents/

  2. 스킬 (Skill) - "도구 상자 / 에이전트의 지식 확장"
   * 정의: 저(Gemini CLI)와 같은 AI 에이전트가 특정 작업을 더 잘 수행할 수 있도록 추가된 '지식과 도구의 패키지'입니다.
   * 위치: packages/tools/skills/



## 🛡️ 1군: 기초 지식 인프라 요원 (The Foundation)
**"NotebookLM의 뇌를 채우고 지식의 토양을 다지는 가장 근본적인 병과"**

| 에이전트 ID | 명칭 | 근본 역할 |
| :--- | :--- | :--- |
| **0_source-fetcher** | **Vault 수집기** | NotebookLM에 부족한 5대 테마 최신 지식을 외부에서 끌어오는 수집의 시작점. |
| **1_source-refiner** | **Gold 선별기** | 수집된 재료를 냉철하게 분석하여 NotebookLM에 저장할 가치가 있는 지식만 남기는 필터. |
| **3_drive-syncer** | **인프라 저장소** | 선별된 지식을 구글 드라이브로 배송하여 NotebookLM과의 물리적 연결을 완성하는 배관공. |

---

## 🏗️ 2군: 응용 영상 생산 요원 (The Application)
**"완성된 지식 토양 위에서 실제 영상 결과물을 만들어내는 정예 병과"**

| 에이전트 ID | 명칭 | 응용 역할 |
| :--- | :--- | :--- |
| **NotebookLM** | **황금 대본가** | 축적된 기초 지식을 바탕으로 시니어 맞춤형 대본을 생성하는 지능형 엔진. |
| **2_video-architect** | **오팔 설계사** | 대본을 Rule 104 규격에 맞춰 15초 단위로 정밀하게 쪼개고 소리를 설계하는 설계사. |
| **2_opal-connector** | **오팔 총감독** | 설계도를 오팔 노드에 직접 투입하고 제작 과정 전반에 작가의 개입을 돕는 현장 감독. |

---

## 🔄 무결점 공정 순서 (Priority Flow)

1.  **[Foundation Start]** `0_source-fetcher` 수집 ➡️ `1_source-refiner` 선별
2.  **[Infrastructure Link]** `3_drive-syncer` 드라이브 저장 ➡️ **NotebookLM 최종 축적**
3.  **[Application Start]** NotebookLM 대본 생성 ➡️ 작가 직접 개입/수정
4.  **[Production]** `2_video-architect` 설계 ➡️ `2_opal-connector` 오팔 노드 투입
5.  **[Completion]** 영상 조각 완성 ➡️ CapCut 온라인 최종 합체

---
*모든 에이전트는 '지식 인프라'가 완벽히 구축된 후에만 '영상 생산' 단계로 진입할 수 있다.*
