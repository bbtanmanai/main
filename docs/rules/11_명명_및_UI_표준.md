# 📋 12. 프론트/어드민 명명 규칙 및 작업 표준 (Naming & UI Standards)

**날짜: 2026년 3월 6일**  
**설계자: 공동감독 김감독 (AI)**  
**승인자: 이감독 (사용자)**

---

## 1. 프론트엔드 명명 규칙 (Next.js 14 App Router)
모든 화면 구성 요소는 `apps/web/src/app/` 폴더 내에 위치하며, 관리자용과 일반 사용자용 페이지를 그룹화하여 관리한다.

| 구분 | 디렉토리 구조 (Route Group) | 주 타깃 | 디자인 시스템 |
| :--- | :--- | :--- | :--- |
| **관리자용** | `app/(admin)/admin/...` | 시스템 관리자 | 다크 테마 기반 어드민 레이아웃 |
| **사용자용** | `app/(front)/...` | 40대 이상 일반 유저 | 시니어 프렌들리 (블루/화이트 고대비) |

### 📂 주요 경로 예시
- `/admin/agent-control`: 에이전트 상태 대시보드.
- `/admin/data-flow`: 전체 데이터 흐름 시각화.
- `/main`: 일반 사용자용 메인 서비스 화면.
- `/mypage`: 사용자 프로필 및 개인 정산 화면.

---

## 2. '도면-시공' 협업 프로세스 (The Blueprint Workflow)
이감독(사용자)의 직관적인 디자인과 김감독(AI)의 기술적 구현을 결합한 표준 공정이다.

1.  **[도면 제출]**: 이감독님이 `design_inbox/` 또는 특정 폴더에 `*.html`, `*.css` 형태로 화면의 시각적 초안(Draft)을 작성한다.
2.  **[기술 변환]**: 김감독(AI)이 도면을 분석하여 `apps/web/src/app/` 하위의 적절한 경로에 **`.tsx`** 파일로 변환 및 시공한다.
3.  **[컴포넌트화]**: 재사용 가능한 UI(GNB, Footer, Buttons)는 `src/components/`로 분리하여 관리한다.

---

## 3. 시니어 접근성 표준 (Senior Accessibility Standard)
모든 프론트엔드 페이지는 다음의 물리적 수치를 엄격히 준수해야 한다.

- **기본 폰트 크기**: 본문 기준 최소 `18px` (필요시 `20px` 이상 권장).
- **버튼 크기 (Touch Target)**: 최소 높이 `48px` 이상 확보.
- **색상 대비**: 텍스트와 배경의 명도 대비는 `4.5:1` 이상을 유지한다 (WCAG AA 등급 준수).
- **직관적 한글**: 기술 용어(Export, API, Logic) 대신 친숙한 표현(내보내기, 공장, 규칙)을 우선 사용한다.

---

## 4. 어드민 페이지 명명 규칙 (Admin Page Convention)
- 모든 어드민 페이지 파일명은 `page.tsx`로 통일하며, 폴더명으로 경로를 정의한다.
- 예: `apps/web/src/app/(admin)/admin/agent-list/page.tsx`
- 페이지 컴포넌트 내부 이름은 반드시 `Admin` 접두어를 붙인다 (예: `export default function AdminAgentListPage()`).

---

## 5. 백엔드 코드 순수성 및 이모지 사용 금지 (Strict Text-only Standard)
시스템의 안정성과 로그 파일의 가독성, 그리고 호환성을 위해 백엔드와 관련된 모든 파일에는 다음 규칙을 적용한다.

- **이모지 사용 절대 금지**: Python 코드(`*.py`), 로그 출력 메시지(`print`, `logging`), JSON 설정 파일 내에서 이모지(Emoji)를 절대 사용하지 않는다.
- **대체 텍스트**: 시각적 강조가 필요한 경우 대괄호`[ ]`나 특수문자(`>>>`, `---`, `*`)를 활용한다.
- **적용 범위**: `apps/api/`, `packages/tools/` 하위의 모든 백엔드 로직에 적용된다.
