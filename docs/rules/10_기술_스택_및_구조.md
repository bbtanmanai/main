**승인자: 이감독 (사용자)**

## 1. 목적 (Goal)
- **Next.js 14 (App Router)** 기반의 현대적인 프론트엔드 아키텍처 구축.
- **FastAPI (Python 3.11+)**를 활용한 고성능 AI 비즈니스 로직 처리.
- **Supabase**를 통한 서버리스 데이터베이스 및 인증 시스템 통합.
- **Deno 사용 금지**: 모든 로직은 Node.js(프론트)와 Python(백엔드)으로 단일화하여 환경 복잡도 제거.

## 2. 핵심 디렉토리 구조 (Core Architecture)
프로젝트는 다음과 같은 **[Multi-App Monorepo]** 구조를 유지하며, 각 폴더의 역할은 엄격히 분리된다.

- **apps/web**: Next.js 14 (TypeScript) 프론트엔드. 시니어 친화적 UI 및 어드민 대시보드 포함.
- **apps/api**: FastAPI (Python) 백엔드. 에이전트 오케스트레이션 및 AI 데이터 브릿지 역할.
- **packages/data**: 모든 정적/동적 데이터 저장소.
    - `raw/`: 수집된 원시 데이터.
    - `refined/`: AI 분석이 완료된 Gold 데이터.
    - `scripts/`: 최종 제작용 대본 데이터 (Rule 104 준수).
- **packages/tools**: `notebooklm-mcp-cli` 등 핵심 유틸리티 패키지.
- **docs/**: 프로젝트의 모든 규칙(rules), 제품 설계(product), 기록(archives) 보관소.

## 3. 현대화 실행 지침 (Implementation Protocol)
1. **의존성 관리**: 파이썬은 `uv`를, 노드는 `npm`을 표준으로 사용하며, 라이브러리 버전은 상호 합의 없이 상향하지 않는다.
2. **에이전트 명칭 통합**: 모든 내부 식별자는 `09_에이전트_설명서.md`의 번호와 명칭(예: `1_source-refiner`)을 따른다.
3. **Zero-API 지향**: 외부 유료 API 호출 전, NotebookLM 내부 기능이나 로컬 정제 로직을 우선 활용한다.
4. **보안 제일**: `.env` 파일은 루트에 단일 관리하며, 민감 정보는 절대 커밋하지 않는다.

## 4. 품질 및 유지보수 표준
- 모든 백엔드 코드는 **Pydantic 모델**을 통해 입출력을 강제 검증한다 (Go 언어 수준의 엄격함).
- 프론트엔드 컴포넌트는 **시니어 접근성 표준**을 준수하여 폰트 및 버튼 크기를 고정한다.
- 매 세션 종료 시, 주요 변경 사항은 `docs/archives/`에 요약 보고한다.
