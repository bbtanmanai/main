    │
    └── 토픽 풀 저장
          Supabase  topic_pool  테이블
            ├── topic:        "혈압에 좋은 음식"
            ├── template_id:  "health-senior"
            ├── viral_score:  8.4
            ├── source_url:   "youtube.com/..."
            └── used:         false


  [topic_pool 테이블]

  CREATE TABLE topic_pool (
    id           UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    topic        TEXT NOT NULL,
    template_id  TEXT NOT NULL,
    viral_score  FLOAT NOT NULL,
    source_url   TEXT,
    used         BOOLEAN DEFAULT FALSE,
    created_at   TIMESTAMPTZ DEFAULT NOW()
  );

  ② 메인 실행 파일 (factory.py) 구조

  """
  scenario_factory/factory.py
  ============================
  실행 방법:
    python factory.py --mode init      # 초기 2,400개 생성
    python factory.py --mode replenish # 재고 보충만
    python factory.py --mode monitor   # 무한 루프 감시
  """

  # 실행 흐름

  def run_init(config):
      """초기 전체 생성 (1회)"""

      # 1. 크롤 DB에서 7점+ 토픽 수집
      topics = topic_collector.fetch(
          min_score=config["topic_source"]["min_viral_score"]
      )

      # 2. 구글 드라이브에 원본 백업
      drive_client.save_topics(topics, folder_id=config["google_drive"]["raw_data_folder_id"])

      # 3. 템플릿 × 스타일 조합으로 시나리오 생성
      for template in config["templates"]:
          for style in config["styles"]:
              target = config["targets"]["initial_count_per_template"]
              # 병렬 생성
              scenarios = scenario_generator.batch_generate(
                  template=template,
                  style=style,
                  topics=topics,
                  count=target // len(config["styles"]),  # 스타일별 균등 분배
                  workers=config["generation"]["parallel_workers"]
              )
              # 수파베이스 저장
              supabase_client.insert_scenarios(scenarios)


  def run_monitor(config):
      """무한 루프 재고 감시 + 자동 보충"""
      while True:
          stocks = supabase_client.get_stock_levels()  # 템플릿별 미사용 재고 수

          for template, count in stocks.items():
              threshold = config["targets"]["min_stock_threshold"]
              if count < threshold:
                  replenish(template, config)

          time.sleep(config["schedule"]["stock_check_interval_minutes"] * 60)


  def replenish(template, config):
      """재고 보충"""
      topics = topic_collector.fetch_latest(template)
      scenarios = scenario_generator.batch_generate(
          template=template,
          count=config["targets"]["replenish_count"]
      )
      supabase_client.insert_scenarios(scenarios)

  ---
  3가지 질문에 대한 답

  ① 구체적 소재 수집 로직 계획안  ✅ 맞음
     → 크롤 DB의 7점+ 데이터에서 토픽 추출 →
       topic_pool 테이블로 관리하는 구조가 필요

  ② 구글 드라이브 → 시나리오 → 수파베이스 파이썬 파일  ✅ 맞음
     → factory.py 단일 진입점으로
       --mode 인자로 init/replenish/monitor 분기

  ③ 사용자 개입 JSON  ✅ 맞음 + 보완 필요
     → 단순 설정 파일이 아니라
       "생산 제어판" 역할을 해야 함
       (템플릿 목록, 재고 임계값, 병렬 수,
        금칙어, 스케줄까지 모두 여기서 제어)

  ---
  한 줄 요약

  소재 수집(topic_collector) → 생성(scenario_generator) → 저장(supabase_client) 3개 모듈을 factory.py가
  production_config.json을 읽어 orchestrate하는 구조입니다. 구현 순서는 production_config.json → supabase_client.py →
  scenario_generator.py → factory.py → topic_collector.py 순이 가장 효율적입니다.
