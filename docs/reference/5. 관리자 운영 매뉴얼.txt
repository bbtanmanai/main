

# 📑 5. 관리자 운영 매뉴얼 (Admin Manual)

## 1. 일일 점검 리스트 (Daily Check)

관리자 대시보드 로그인 후 가장 먼저 확인해야 할 지표입니다.

* **신규 매출 현황**: 어제 발생한 총 매출액 및 결제 수단별 비중 확인.
* **취소/환불 요청**: 시니어 유저의 환불 요청 건수를 확인하고, 콘텐츠 열람 여부 체크 후 승인/거절 처리.
* **비정상 활동 탐지**: 동일 IP에서 단시간 내 다량 결제가 일어나는지, 또는 조직도상의 비정상적인 Spillover 배치가 있는지 모니터링.

## 2. 주간 정산 프로세스 (Weekly Settlement)

가장 중요한 **'수요일 지급'**을 위한 단계별 절차입니다.

| 단계 | 요일 | 활동 내용 |
| --- | --- | --- |
| **1단계** | 월요일 | **대상 확정**: 3주 전 매출 중 `status`가 `confirmed`(확정)인 데이터 필터링. |
| **2단계** | 화요일 | **지급 명세서 생성**: 유저별 직접 수당 + 후원금 합산 및 3.3% 원천징수 계산. |
| **3단계** | **수요일** | **이체 실행**: 펌뱅킹 연동 또는 엑셀 대량 이체를 통해 실제 입금 처리 후 `paid` 상태로 변경. |

## 3. 회원 및 조직 관리 (User & Matrix Management)

* **Spillover 수동 조정**: 시스템 오류나 특수한 사정으로 인해 조직 배치가 잘못된 경우, `referrals` 테이블의 `parent_id`를 수정하여 위치를 조정합니다.
* **작가 등급 관리**: 일정 매출액을 달성한 유저의 등급(Role)을 수동 혹은 자동(Batch)으로 상향 조정합니다.
* **정산 계좌 승인**: 유저가 등록한 통장 사본과 실명이 일치하는지 확인 후 '정산 가능' 상태로 승인합니다.

## 4. CS(고객 서비스) 대응 가이드

시니어 타겟 서비스의 특성상 발생하는 주요 문의 대응법입니다.

* **"돈이 왜 안 들어오나요?"**: `scheduled_payout_date`를 조회하여 "다음 주 수요일에 입금될 예정입니다"라고 친절히 안내합니다.
* **"비밀번호를 잊어버렸어요"**: 관리자 권한으로 임시 비밀번호를 발급하거나 본인 확인 후 초기화 링크를 발송합니다.
* **"결제 수단을 바꾸고 싶어요"**: 기존 건 취소 후 재결제를 안내하거나, 카드사 정보를 확인해 줍니다.

## 🛠️ 관리자 페이지(Admin UI) 구현 프롬프트

> **"관리자 전용 정산 대시보드를 생성해줘:"**
> 1. 이번 주 수요일에 지급해야 할 **총 유저 수와 전체 지급액**을 상단에 요약 표시해줘.
> 2. 버튼 하나로 **'이번 주 정산 대상자 엑셀 다운로드'** 기능이 작동하게 해줘.
> 3. 각 지급 내역 옆에 **[지급 완료 처리]** 버튼을 두어 클릭 시 `actual_payout_at` 시간이 기록되게 해줘.
> 4. 수당 종류(`direct` vs `upline_bonus`)별로 필터링해서 볼 수 있는 기능을 추가해줘.
>

### 🛠️ 관리자 페이지 내 '정산 모듈' 필수 기능

#### 1. 수요일 지급 리스트 자동 생성 (Batch Processing)

* **기능**: 매주 월요일, 시스템이 자동으로 3주 전 매출 데이터를 긁어모아 **"이번 주 수요일에 돈을 줄 사람 명단"**을 만듭니다.
* **데이터**: [사용자명 / 은행 / 계좌번호 / 지급액(3.3% 공제 후) / 직접+후원 합계]
* **UX**: 관리자는 이 리스트를 눈으로 확인하고 '지급 승인' 버튼만 누르면 됩니다.

#### 2. 은행 이체용 엑셀 다운로드 (Mass Pay Export)

* **기능**: 승인된 리스트를 클릭 한 번으로 **기업은행, 신한은행 등 각 은행의 '대량 이체 양식(Excel/CSV)'**에 맞춰 다운로드합니다.
* **효과**: 관리자는 이 파일만 인터넷 뱅킹에 업로드하면 수백 명에게 동시에 돈을 보낼 수 있습니다. (별도의 고가 펌뱅킹 연동 없이도 가능)

#### 3. 세무 신고용 로우 데이터 추출

* **기능**: 매달 말, 혹은 종합소득세 신고 시기에 맞춰 **원천징수(3.3%) 내역**을 엑셀로 뽑아냅니다.
* **효과**: 세무사에게 이 파일만 전달하면 세무 처리가 끝납니다.

### 🏗️ 클로드 코드(Claude Code)로 구현할 정산 로직

안티그래비티와 클로드 환경에서 이 '정산 프로그램' 기능을 구현할 때 아래 프롬프트를 사용하세요.

> **"정산 관리자 모듈 상세 구현:"**
> 1. `earnings` 테이블에서 `status`가 'confirmed'이고 `scheduled_payout_date`가 이번 주 수요일인 데이터를 모두 불러오는 **'이번 주 정산 관리'** 화면을 만들어줘.
> 2. 총 지급액과 인원수를 상단에 요약해줘.
> 3. 선택한 유저들의 정보를 **은행 대량 입금 양식(CSV)**으로 내려받는 기능을 추가해줘.
> 4. 지급 완료 후 버튼을 누르면 해당 데이터들의 상태를 'paid'로 일괄 변경하고 실제 지급 시각(`actual_payout_at`)을 기록해줘.
>
