


## 1. 전체 구조 개요


> [!1. 입력]
>     - 시나리오 전체(또는 에피소드/씬 단위 텍스트)
>     - 배우 2명의 캐릭터 시트 이미지(얼굴+전신, 여러 장 가능)

> [!2. 내부 처리]
>     - 시나리오 → “샷/프레임 단위”로 자동 분할
>     - 각 프레임마다 필요한 캐릭터, 상황, 카메라, 분위기 추출
>     - 참조 이미지(캐릭터 시트)와 매칭해서 “캐릭터 외형 고정” 규칙 반영
>     - 결과를 “이미지/비디오 생성용 상세 프롬프트” JSON으로 출력

> [!3. 출력]
>     - 프레임별 상세 프롬프트 (예: core / extended 두 버전)
>     - 한국어+영어 동시 출력 옵션


---


> [!1. 모델/플랫폼 선택]
> - 플랫폼: Google AI Studio → Gemini API 사용
> - 모델 : gemini-2.0-pro` (멀티모달, 시나리오+이미지 동시에 입력)​
> - 응답 형식:  response_mime_type="application/json"`으로 고정해서, 항상 JSON만 받도록 설정​

## 2. 프롬프트 엔진(시스템 인스트럭션) 설계

앱에서 Gemini에 보낼 “고정 시스템 프롬프트”를 하나 만들어 두는 것이 핵심입니다.​

## (1) 역할/제한 규칙

예시(요약):

- 너는 “영상/이미지 생성용 텍스트 프롬프트 엔진”이다.
- 항상 JSON 포맷의 텍스트 프롬프트만 출력한다.
- 입력으로 받는: ⓐ 시나리오 텍스트 ⓑ 2명의 배우 캐릭터 시트 이미지
- 출력: 시나리오를 프레임 단위로 나눈 후, 각 프레임에 대해 “캐릭터 외형, 의상, 표정, 포즈, 카메라, 배경, 분위기”가 포함된 상세 프롬프트를 JSON 배열로 출력.
- 캐릭터 관련 규칙(당신이 이미 정한 3원칙 반영)
    1. 이미 등장한 캐릭터의 얼굴/체형/헤어/의상 콘셉트를 바꾸지 않는다.
    2. 전체 화풍과 톤을 유지한다.
    3. 물리적 시간·공간·인체에 대해 과장된/불가능한 묘사를 하지 않는다.

---

## (2) 출력 포맷 정의

앱에서 파싱하기 쉽게, JSON 스키마를 먼저 정해 둡니다.

예시:

json

```
{
  "frames": [
    {
      "frame_id": "S1_I001",
      "short_summary_ko": "관우 등이 드러난 채 화타가 상처를 살피는 첫 장면",
      "prompt_ko": "… (이미지 전용 프롬프트)",
      "prompt_en": "…",
      "camera": {
        "shot_size": "medium close-up",
        "angle": "slightly low angle"
      },
      "style": {
        "lighting": "warm single lantern on the left, strong contrast",
        "overall_tone": "dark, tense, realistic historical photograph"
      },
      "characters": [
        {
          "name": "관우",
          "from_reference_image": "character_GuanYu_main",
          "description_for_model": "…"
        }
      ]
    }
  ]
}
```

이 구조를 시스템 프롬프트에 “반드시 이 형식 그대로 출력하라”라고 명시합니다.

## 3. 입력 단계 설계

## (1) 시나리오 입력

- UI에서:
    
    - 프레임별 마스타코드 번호를 자동입력한다
    - 마스터코드 규칙 : 
    - sXXcXXbXX  9자리로 제한한다.
    - XX : 00 ~ 99 두자리 정수로 한다.
    - sXX : 사용자 영역, 변형 시키지 말아라
    - cXX : 컷별, 장면전환
    - bXX : 비디오 영역, 카메라 이동 등
    - 주로 c 코드 위주로 마스터코드를 변형을 주면된다
    - 첫 요청: “마스터코드별 시나리오 → 마스터코드 리스트(JSON)”만 뽑기
    - 둘째 요청: 마스터코드별로 프레임 프롬프트 생성 (체인 프롬프트)​ 

## (2) 캐릭터 시트 이미지 입력

- 등장 하는 캐릭터별로:
    - 얼굴 클로즈업 1–3장
    - 상반신/전신 1–3장
    - 기본/전투/일상 의상 각 1장 정도를 추천​
    
- 앱 내부에서:
    
    - Google AI Studio / Gemini API의 파일 업로드 기능 사용​
    - 업로드 결과로 받는 file id를

> [!NOTE]
>     character_GuanYu_main`, `character_HuaTuo_main`처럼 매핑해서 시스템 프롬프트에 “이 이미지가 이 캐릭터의 기준 얼굴/체형이다”라고 알려줌.


Gemini 호출 시:

- `contents = [ 시나리오 텍스트, 캐릭터 이미지 1, 캐릭터 이미지 2, …, 사용자 지시 ]` 식으로 멀티모달 입력    

---

## 4. “프레임 단위 분해” 로직

   
1단계: 마스터코드 리스트
2단계: 프롬프트 생성

- 각 샷을 다시 모델에 넣으면서:
    
    - “이 샷 하나만 기반으로, 위의 프롬프트 JSON 형식대로 frame을 1~3개로 세분화해라. 캐릭터 외형은 참조 이미지를 기준으로 자세히 묘사해라.”

이 체인 구조는 공식 전략에도 나와 있는 “prompt chaining” 패턴에 해당합니다.​

---

## 5. 캐릭터를 “최대한 참조”하게 만드는 방법

필수 포인트만 요약하면:

1. 항상 이미지와 함께 호출
    
    - `contents=[캐릭터 시트 파일, "이 이미지의 인물 외형을 자세히 서술하고, 이 스타일을 유지하면서 아래 시나리오 장면용 프롬프트를 만들어라.", 시나리오 일부]`​
        
2. 언어로 다시 한 번 외형 고정
    
    - 예: “The warrior must have the same face, beard, hairstyle, and body type as the man in the reference image. Do not change his age, ethnicity, or overall build.”​
        
3. 금지 규칙을 명시
    
    - “Do not invent new tattoos, scars, or accessories that are not in the reference image, unless the script explicitly requires a wound or object.”
        
4. 샷마다 캐릭터 요약 재사용
    
    - 한 번 뽑은 캐릭터 설명(텍스트)을 저장해 두고, 이후 요청에서 항상 함께 넣기:
    - “Use the following character bible as ground truth and do not contradict it: {GuanYu_description, HuaTuo_description}.”

## 6. 구현 기술 스택/흐름 예시

## (1) 백엔드

- 언어: Python 또는 Node.js
    
- Google AI Studio → Gemini API 연동[google+1](https://ai.google.dev/gemini-api/docs/imagen)
    
    - Python 예:
        
        - `google-genai` 클라이언트 사용
            
        - 파일 업로드 → file id 획득 → `generate_content` 호출
            

대략 구조:

1. `/upload-characters`
    
    - 이미지 업로드 → Gemini Files API에 올리고 file id 저장

    - 시나리오 텍스트 → Gemini 호출 → 씬/샷 리스트 JSON으로 저장
        
3. `/generate-prompts`
    
    - 시나리오의 특정 씬/샷 + 캐릭터 file id + 캐릭터 바이블 텍스트 → 프레임별 프롬프트 JSON 생성
        

## (2) 프론트엔드

- 간단히 시작하려면:
    
    - Streamlit / Gradio / Next.js + React 중 편한 것
        
- 화면 구성:
    
    1. 시나리오 입력 창(텍스트 붙여넣기)
        
    2. 캐릭터 시트 업로드 영역(배우 A, B)
        
    3. “씬 분해” 버튼 → 씬/샷 리스트 테이블 표시
        
    4. 씬/샷 선택 후 “프롬프트 생성” 버튼
        
    5. 결과 프레임 리스트 + 각 프레임의 core/extended 프롬프트 보기/복사 기능

# 시나리오 기반 프레임별 이미지 프롬프트 앱 설계 (Google AI Studio / Gemini)

## 1. 목표

- 시나리오 텍스트와 2명의 배우 캐릭터 시트 이미지를 입력하면  
  **각 프레임(컷)마다 이미지 생성용 상세 프롬프트**를 자동으로 만들어 주는 앱.
- 결과물은 **텍스트 프롬프트만** 출력하며, 이미지/비디오를 직접 생성하지 않는다.
- 사람(사용자)이 중간에 프롬프트를 검토·수정할 수 있는 워크플로우를 포함한다.

---

## 2. 전체 워크플로우 개요

1. 입력 단계
   - 시나리오 텍스트 (씬/샷 단위)
   - 배우 2인의 캐릭터 시트 이미지 (얼굴, 전신 등 여러 장)
1. 분석 단계 (Gemini API)
   - 시나리오를 **씬 → 프레임(이미지 단위)**로 분해
   - 각 프레임마다 등장 인물, 상황, 구도, 감정, 배경, 스타일을 추출
1. 초안 생성 단계
   - Gemini가 프레임별 **이미지 프롬프트 초안(draft)**을 JSON 형식으로 생성
1. 중간 점검 단계 (사용자 UI)
   - 엑셀/테이블 형태 화면에서 프레임별 프롬프트를 검토·수정·확정
1. 출력 단계
   - 확정된 프롬프트를 JSON/CSV/엑셀로 내보내어  
     외부 이미지 생성 모델(Imagen, 기타 이미지 AI)에 사용

---

## 3. 모델/플랫폼 선택

- 플랫폼: **Google AI Studio → Gemini API**.
- 모델:
  - 멀티모달 이해용: `gemini-2.0-flash` 또는 `gemini-2.0-pro` (텍스트+이미지 입력).
- 사용 방식:
  - Gemini는 **“프롬프트 텍스트 초안 생성기”**로만 사용한다.
  - 최종 이미지는 Imagen 등 별도 이미지 생성 모델에서 생성.

---

## 4. 시스템 인스트럭션(역할 정의)

Gemini 호출 시 고정으로 넣는 시스템 프롬프트 개념.

### 4.1 역할

- 너는 “시나리오 기반 이미지 프롬프트 엔진”이다.
- 입력:
  - 시나리오 텍스트
  - 2명의 캐릭터 시트 이미지 (배우 외형 레퍼런스)
- 출력:
  - 시나리오를 프레임 단위로 나누고,
  - 각 프레임마다 이미지 생성용 프롬프트를 **JSON 형식**으로만 출력한다.
- 이미지나 비디오를 생성하거나, 마크다운/자연문을 따로 쓰지 않는다.  
  오직 JSON만 출력한다.

### 4.2 캐릭터/스타일 규칙

사용자가 이미 정의한 3원칙을 반영.[cite:6][cite:8][cite:10]

1. 이미 등장한 캐릭터의 얼굴, 체형, 헤어, 기본 의상 콘셉트를 바꾸지 않는다.
2. 전체 화풍과 스타일(조명, 색감, 분위기)을 유지한다.
3. 물리적 시간·공간·인체에 대해 과장되거나 불가능한 묘사를 하지 않는다.

추가 규칙:

- 각 프롬프트에서 캐릭터는 **캐릭터 시트 이미지와 일관되게** 묘사한다.
- 새 문신·흉터·장신구를 멋대로 만들지 않는다 (시나리오에 명시된 경우만 예외).[cite:8][cite:9]
- 이미지 한 장에만 해당하는 내용으로 묘사하고, 카메라 움직임이나 시간 경과 표현은 피한다.[web:16][web:25]
