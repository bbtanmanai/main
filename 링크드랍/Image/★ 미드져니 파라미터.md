```

[매체 & 스타일] (Medium & Style)

가장 먼저 정의해야 전체 톤이 잡힙니다.
예: 지브리 애니메이션 스타일, 수채화 일러스트, 1990년대 셀 애니메이션

[주체 & 외형] (Subject & Appearance)
```

이미지 무게  (  -- iw  )는 이미지 프롬프트가 동반되는 
텍스트 프롬프트에 비해 얼마나 많은 영향을 미치는 매개변수입니다.


0 : 이미지 영향 최소화 3 : 이미지 영향 최대화
iw역할: 첨부한 이미지를 텍스트 프롬프트보다 더 중요하게 여깁니다.

단점: 프롬프트로 "춤추는 장면"이라고 써도, 원본이 "앉아 있는 장면"이고

-iw 3   을 주면 AI가 고집을 부려 그냥 앉혀버릴 확률이 높습니다.

#### ★ 이미지를 앞에 첨부 시 필수 : — iw 3 이미지를 따라한다.

#### ★ 이미지를 sref 뒤에 첨부 시 필수 : — sw 1000 이미지의 스타일을 따라한다.



#### ==★ 이미지 프롬프트 / 스타일 프롬프트 / 옴니 래퍼런스==

#### 옴니 레퍼런스 (Omni Reference)
역할: AI에게 “이걸 기억해”라고 명령하는 기능
장점: 장기적 일관성, 시리즈 제작에 강함
단점: 강도 조절 실패 시 결과물이 단조로워질 수 있음
키워드: “기억해, 이 캐릭터”

#### 1. 이미지 프롬프트 (Image Prompt)
역할: 스타일, 분위기만 대충 참고
장점: 자유롭고 빠름
단점: 일관성 없음, AI는 기억 안 함
키워드: “이 느낌 알아? 그거 해줘”

#### 2. 스타일 프롬프트 (Style Prompt)
역할: 반복적인 스타일링을 빠르게 적용
장점: 스타일 유지
단점: 캐릭터 고정 불가, 외형 재현 X
키워드: “이런 스타일로 꾸며줘”

요약하자면:

> [!NOTE]
> #### 계속 같은 인물, 같은 스타일로 그리고 싶다” = 옴니 레퍼런스
> #### 즉흥적이고 스타일리시한 변주를 하고 싶다” = 이미지 프롬프트

#### 주의할 점
Omni Weight(기억 강도)를 너무 세게 주면 창의성이 사망함.
원본 이미지가 너무 복잡하거나 배경이 과하면 혼선 생김.
표정, 포즈까지 고정하고 싶으면 텍스트 프롬프트랑 같이 세팅해야 함.


```
-- quality <0.25, 0.5, 1> (또는 --q)
```
​기능: 이미지 생성에 소요되는 시간과 품질을 결정합니다
0.25: 빠른 생성, 낮은 디테일 0.5: 균형잡힌 생성 속도와 품질 1: 느린 생성, 높은 디테일 (기본값)

```
-- seed <number>
```
기능: 특정 시드 값을 사용하여 일관된 이미지 생성
같은 시드와 프롬프트로 유사한 구성의 이미지를 재생성할 수 있습니다


```
-- style raw
```
"미드저니 특유의 느낌" 제거     
Raw 모드 (  -style raw  ): 그런 꾸밈을 최소화하고, 담백하고 솔직하게 그립니다.
사진으로 치면 필터 없는 원본 카메라 느낌입니다. / 프롬프트 충실도 상승
지브리 스타일은 그 자체로 예술적인 감성과 예쁜 색감이 중요합니다.

```
-- style raw    
```
를 쓰면 그림이 너무 투박해지거나, 우리가 원하는 몽환적인 수채화 느낌이 없어짐

```
-- niji 6    
```
​기능: Niji 모델 6를 사용하여 애니메이션 스타일 이미지 생성
일본 애니메이션, 만화 스타일에 최적화되어 있습니다

```
-- stylize <number> (또는 --s)
```
예술적 강도 표현. 낮은 값 (0-100): 프롬프트에 충실 덜 예술적
기본값 (100): 균형잡힌 스타일화
높은 값 (100-1000): 매우 예술적이고 창의적인 해석
동화책 삽화 느낌을 원하시면 -s 250 정도

```
-- weird <0-3000> (또는 --w)
```
기능: 이미지에 기발하고 실험적인 요소를 추가합니다
예상치 못한 색감, 구성, 분위기를 만들어냅니다
높은 값일수록 더욱 독특하고 초현실적인 결과

```
--fast, --turbo, --relax
```
기능: 이미지 생성 속도 모드를 선택합니다
--turbo: 가장 빠른 생성 (추가 요금)
--fast: 빠른 생성 (기본 모드)
--relax: 느린 생성 (구독 플랜에 따라 무제한)


```
-- no <object>
```
기능: 이미지에서 특정 요소를 제외합니다
사용 예시: --no people, --no trees, --no buildings


```
-- tile
```
기능: 반복 패턴으로 사용할 수 있는 타일 이미지를 생성합니다
텍스처, 배경, 패턴 디자인에 유용합니다

```
-- stop <10-100>
```
기능: 이미지 생성 프로세스를 중간에 중단합니다
낮은 값: 더 추상적이고 덜 완성된 이미지
높은 값: 거의 완성에 가까운 이미지


```
--repeat <1-40> (또는 --r)
```
기능: 하나의 프롬프트로 여러 장의 이미지를 동시에 생성합니다
빠르게 다양한 변형을 확인할 수 있습니다

```
-- video
```
기능: 이미지 생성 과정을 동영상으로 기록합니다
생성 후 이모지 반응으로 동영상 링크를 받을 수 있습니다


스타일 + 메인 주제/장면 + 효과 + 분위기/무드 + 배경 설정 + 기술적설정

Upscale (Subtle)
(미세한 업스케일): 원본 이미지의 느낌을 최대한 유지하면서 해상도를 높입니다.
디테일 변화를 최소화하고 싶을 때 사용합니다.

Upscale (Creative)
(창의적 업스케일): 해상도를 높이면서 AI가 창의적인 디테일을 추가하거나
텍스처를 재해석합니다. 원본과 분위기가 조금 달라질 수 있습니다.

Redo Upscale (Subtle): 
현재 이미지의 형태를 최대한 보존하면서 해상도를 높여 깔끔하게 만듭니다.

Redo Upscale (Creative): 
해상도를 높이는 과정에서 AI가 이미지에 어울리는 디테일(피부 질감, 수염의 세밀함 등)을 창의적으로 덧붙입니다.

Vary (Subtle)
(미세한 변형): 현재 이미지와 매우 유사하지만, 약간의 디테일(표정, 털의 결 등)만
다른 4개의 새로운 이미지를 생성합니다.

Vary (Strong)
(강한 변형): 현재 이미지의 핵심 컨셉은 유지하되, 구도나 스타일을 더 과감하게 바꾼
4개의 새로운 이미지를 생성합니다.

Vary (Region)
(영역 수정): 이미지의 특정 부분만 선택해서 다시 그리도록 지시합니다.
(예: 샤미의 눈 색깔만 바꾸거나, 꼬리 모양만 수정할 때 사용)

Zoom Out 2x / 1.5x: 
카메라를 뒤로 빼는 효과입니다. 캐릭터는 작아지고 주변 배경(삼국지의 전쟁터나 막사 등)이 더 넓게 그려집니다.

Custom Zoom: 
원하는 배율을 직접 숫자로 입력하여 배경을 확장합니다.

Make Square:
가로로 긴 이미지를 1:1 정사각형 비율로 맞추고 남는 공간을 배경으로 채웁니다.

화살표 버튼 (←, →, ↑, ↓): 
이미지를 선택한 방향으로 이어 붙여서 더 길게 그립니다. 예를 들어, 오른쪽 화살표를 누르면 유비의 옆에 서 있는 관우나 장비를 그릴 공간이 생깁니다.

재생성 원본 이미지의 느낌을 최대한 유지하면서 해상도를 높입니다.
