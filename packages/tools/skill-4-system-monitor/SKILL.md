# 🛠️ 시스템 감시 에이전트 (System Monitor)

## 개요
프로젝트 세션의 대화 내용 및 로그를 실시간으로 모니터링하여 데이터의 무결성을 유지하고 프로젝트의 연속성을 보장하는 독립형 감시 스킬입니다.

## 핵심 기능
- **침묵 감지 및 요약 (Auto-Patch)**: 10분 이상 활동이 감지되지 않을 때, 마지막 세션의 주요 내용을 자동으로 요약하여 패치 파일(`.md`)로 생성합니다.
- **일일 로그 아카이빙 (Nightly Archiver)**: 매일 밤 11시 59분에 하루 동안의 대화 기록을 `docs/archives` 폴더로 이동하여 보관합니다.
- **문서화 지원**: 프로젝트의 규칙이나 공정 우선순위가 변경되었을 때, 이를 자동으로 기록하여 지식 부채를 방지합니다.

## 사용법 (AI 지침)
- 세션이 종료되기 전이나, 장기간 휴식 후 복귀했을 때 생성된 `PATCH_*.md` 파일을 확인하여 이전 문맥을 빠르게 파악하세요.
- 시스템의 안정성과 로그 관리를 위해 본 스킬을 백그라운드 프로세스로 가동할 수 있습니다.

## 제약 사항
- 요약 기능을 위해 OpenAI API 키(`OPENAI_API_KEY`)가 필요합니다.
- 감시 대상 로그 파일의 경로가 올바르게 설정되어 있어야 합니다.
