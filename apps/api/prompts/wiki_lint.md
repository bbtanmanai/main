## 검수 항목 5가지

### 1. 대본 퀄리티 검수
각 챕터 대본을 위 문체 헌장 6원칙 기준으로 점검한다:
- 첫 문장이 충분히 강한가? (독자가 멈추는가?)
- 중간 이탈 구간이 있는가? (감정 흐름이 평탄한 구간)
- 감정 단어를 직접 사용한 문장이 있는가? ("불안했다", "슬펐다" 등 — 원칙 1 위반)
- 같은 종결어미가 3회 연속 사용된 구간이 있는가? (원칙 6 위반)
- 각 장면의 끝에 훅이 있는가? (원칙 5)
- 대사 비율이 40% 이상인가?
- 비밀 인물의 행동 단서가 2개 이상 심어졌는가? (원칙 3)

### 2. 정보 신뢰성 평가
facts.md 내 팩트들에 대해:
- 출처가 명시되어 있는가?
- 출처가 신뢰할 만한가? (공식 기관, 학술 자료 vs 불명확한 출처)
- 수치/날짜가 구체적인가?

### 3. 지식 충돌 점검
위키 전체 페이지를 검토해:
- 같은 사실을 다르게 기술한 페이지가 있는가?
- 오래된 페이지(updated_at 기준)의 내용이 최신 챕터와 모순되는가?
- 인물의 나이·직업·관계가 페이지마다 일관되는가?

### 4. 고아 페이지 & 누락 링크 탐지
- 다른 페이지에서 한 번도 링크되지 않은 페이지 목록
- 본문에서 언급되었으나 `[[링크]]`가 없는 인물/소품/개념
- 중요 개념인데 독립 페이지가 없는 항목

### 5. 복선 추적
foreshadows.md 기준:
- 심어졌으나(planted) 아직 회수(recovered)되지 않은 복선 목록
- 회수했지만 foreshadows.md에 기록이 안 된 복선
- 챕터 진행상 회수되어야 할 시점이 지난 복선

## 출력 형식
아래 마크다운 구조로 lint_report.md를 생성한다:

```markdown
# Lint Report — {series_title}
생성일시: {datetime}

## 1. 대본 퀄리티
### 🔴 수정 필요
- [챕터 N] {위반 원칙} — {문제 문장} → 수정안: ...
### 🟡 주의
- ...
### 🟢 양호
- ...

## 2. 정보 신뢰성
| 팩트 | 출처 | 신뢰도 |
|------|------|--------|

## 3. 지식 충돌
| 페이지 A | 페이지 B | 충돌 내용 |
|---------|---------|---------|

## 4. 고아 페이지 & 누락 링크
- 고아 페이지: [[...]]
- 누락 링크: ...

## 5. 미회수 복선
| 복선 내용 | 심어진 챕터 | 예상 회수 시점 | 상태 |
|-----------|-------------|---------------|------|

## 권장 조치
1. ...
```

보고서 출력 후 즉시 수정 가능한 항목(링크 추가, 충돌 해소)은 patch JSON도 함께 출력한다.
