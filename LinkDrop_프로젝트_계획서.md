## 2. 문제 정의 — 왜 만드는가

### 2-1. 시장 현황

- 국내 시니어(50세 이상) 인구 **약 1,700만 명**, 스마트폰 보급률 90% 이상
- 그러나 콘텐츠 수익화 시장은 **복잡한 학습 과정과 기술 장벽**이 전제
- 기존 '온라인 부업' 시장은 강의 중심 → 수익보다 **배움에 시간·돈을 먼저 소비**하게 설계

### 2-2. 고객 페인 포인트

| 페인 포인트 | 구체 증상 |
|-----------|---------|
| 진입 장벽 | 영상 편집 툴·글쓰기 플랫폼 러닝커브가 높아 시작조차 못 함 |
| 학습 선행 강요 | 강의 먼저 → 수익은 나중 → 중도 포기율 극히 높음 |
| 기술 지원 부재 | 오류 발생 시 해결 경로 없음 |
| 신뢰 부재 | 결제 후 실질 보상이 언제·어떻게 오는지 불투명 |
| 디지털 소외 | UI가 젊은 세대 기준으로 설계 — 폰트 작고 버튼 작음 |

### 2-3. LinkDrop의 핵심 명제

> **"₩59,000을 낸 모든 소비자를 AI 원데이클래스 강사 수준으로 만드는 것이 우리의 교육 목표다."**  
> 전자책은 단순 정보 판매가 아니다. 구매 순간부터 4단계 교육 경로가 시작되며,  
> 그 종착점은 자신이 배운 것을 남에게 가르칠 수 있는 **강사 역량**의 완성이다.

## 3. 솔루션 개요 — 무엇을 만드는가

LinkDrop V3는 **세 개의 레이어**가 순서대로 맞물리는 교육 기반 수익화 플랫폼입니다.

### Layer 1 — 전자책 (교육의 문)

- **9개 랜딩 페이지**: 시니어가 관심 가질 온라인 부업 주제별로 각각 설계
- **₩59,000 전자책**: 단독으로도 충분한 가치 — 지식·실행 가이드·보너스 금고 8종 포함
- **교육 목표 내재화**: 구매 즉시 "강사가 될 수 있다"는 비전을 심는 것이 1차 목적

### Layer 2 — V3 툴 (교육의 실습실)

- **웹소설 자동 생성**: 주제·세계관 입력 → Gemini AI가 6챕터 연재 대본 자동 생성
- **영상 자동 생산**: 대본 → 씬·컷 분절 → 이미지 프롬프트 → 키프레임 영상 자동 조립
- **YouTube + 네이버 이중 게재**: 완성 콘텐츠를 두 플랫폼에 자동 업로드
- **역할**: 소비자가 "직접 만들어보는" 실습 경험을 제공하여 강사 역량을 내재화

### Layer 3 — 강사 양성 (교육의 완성)

- **AI 원데이클래스 강사 인증**: V3 실습 경험을 바탕으로 남을 가르칠 수 있는 수준으로 공인
- **완성형 강의 Kit 제공**: 강사는 커리큘럼을 만들지 않아도 됨 — 실행만 하면 됨
- **자기증식 루프**: 강의 수강생이 새로운 ₩59,000 소비자로 자동 유입

### 세 레이어의 관계

```
Layer 1 전자책       Layer 2 V3 툴        Layer 3 강사 인증
────────────────    ────────────────    ────────────────
"왜 해야 하는가"  →  "어떻게 만드는가"  →  "남에게 가르치기"
개념·동기 부여       직접 체험·실습       역량 공인·수익화
₩59,000 일회성       무료 체험 후 유료     강의비 + 수당
```

세 레이어가 순서대로 완성될 때, 소비자는 강사가 되고 강사는 새로운 소비자를 만들어냅니다.

## 4. 비즈니스 모델 — 어떻게 수익을 내는가

### 4-1. 플랜 구성

| 플랜 | 금액 | 제공 내용 | 교육 단계 |
|------|------|---------|---------|
| **스타터** | ₩59,000 (1회) | 전자책 + 8종 보너스 금고 + V3 무료 체험 7일 + 강사 양성 과정 입장권 | 1~2단계 (학습·실습) |
| **강사 툴** | ₩29,000/월 | V3 웹소설·영상 무제한 생성 — 강의 시 실시간 시연 전용 | 3~4단계 (내재화·강사) |

### 4-2. 파트너 수당 구조

> 방문판매법 제22조 준수 — 수당 합계 VAT 제외 공급가액의 **35% 이내**

| 수당 유형 | 지급액 | 비율 | 조건 |
|---------|------|------|------|
| 스타터 직접 판매 수당 | ₩18,700 | 34.9% | 강사가 수강생을 유치하여 구매 완료 시 1회 |
| 강사 툴 후원 수당 | ₩9,200/월 | 34.9% | 유치한 수강생이 강사 툴 구독 유지 중 매월 *(법무 검토 중)* |

**정산 프로세스**

```
1주차 → 매출 집계 (월~일 00:00~23:59)
2주차 → 취소·환불 필터링 및 최종 지급액 확정
3주차 수요일 → 등록 계좌로 수당 일괄 지급 (3.3% 사업소득세 원천징수 후)
```

### 4-3. 법적 구조

- **법적 지위**: 후원방문판매 (방문판매법 준수)
- **수당 단계**: 직근 추천인 **1인에게만** 지급 — 2단계 이상 전파 없음 (다단계판매 방지)
- **스필오버(Spillover)**: 3xN 매트릭스 구조 — 하위 3자리 초과 시 빈 자리 자동 배정

## 5. 판매 퍼널 — 어떻게 고객을 만드는가

### 5-1. 핵심 전략: 9개 니즈 타겟 랜딩 페이지

고객의 니즈를 사전에 알 수 없으므로, **시니어가 관심 가질 9개 온라인 부업 주제**를 각각의 랜딩 페이지로 설계합니다. 방문자는 자신과 공감하는 페이지에서 구매를 결정합니다.

| # | 랜딩 주제 
|---|---------|---------|------------|
| 1 | **웹소설 작가로 월 수익 만들기** 
| 2 | **내 인생 이야기가 베스트셀러가 된다** 
| 3 | **유튜브 쇼츠로 매달 광고비 받기** 
| 4 | **내 전문 지식을 영상 강의로 팔기** 
| 5 | **블로그 글 하나로 매달 광고 수익**
| 6 | **재테크 정보 채널로 부수입 만들기** 
| 7 | **건강·다이어트 정보로 돈 버는 법** 
| 8 | **1인 기업이 마케팅, CS 자동화로직을 구성 하는 방법** 
| 9 | **손주·육아 꿀팁으로 인플루언서 되기**

## 6. AI 원데이클래스 강사 양성 프로그램

> **핵심 교육 목표**: ₩59,000을 낸 소비자를 단순 구매자로 끝내지 않는다.  
> **AI 원데이클래스를 직접 운영할 수 있는 강사 수준**으로 만드는 것이 이 플랫폼의 존재 이유다.

### 6-1. 왜 강사 양성인가 — 교육 철학

기존 AI 교육 시장의 문제는 **"배웠지만 써먹지 못한다"** 는 것입니다. LinkDrop은 반대로 설계합니다. 툴을 직접 써서 결과물을 만들고, 그 결과물로 강의를 하고, 강의를 통해 수강생에게 같은 경험을 전달합니다. **경험이 곧 교육 자산**이 됩니다.

```
₩59,000 소비자
    ↓  1단계: 전자책으로 개념 학습
    ↓  2단계: V3 툴로 AI 콘텐츠 직접 생성
    ↓  3단계: 리셀러로 판매 경험 축적
🎓 AI 원데이클래스 강사
    ↓  "내가 ₩59,000으로 여기까지 왔다"가 강의 핵심 스토리
강의 수강생 → 새로운 ₩59,000 소비자 유입
    ↓
LinkDrop 생태계 자기증식 (Flywheel)
```

강사 1명이 원데이클래스에서 수강생 10명을 모집하면 → 최대 10건의 신규 판매 = **₩187,000 신규 수당** 발생. 강사는 LinkDrop의 가장 강력한 오프라인 영업·교육 조직입니다.

### 6-2. 강사 성장 4단계 경로

| 단계 | 명칭 | 달성 조건 | 획득 권한 |
|------|------|---------|---------|
| **1단계** | 학습자 | ₩59,000 스타터 구매 | 전자책 + 강사 양성 과정 입장권 |
| **2단계** | 실습자 | V3 체험 기간 내 콘텐츠 1편 이상 완성 | 리셀러 파트너 링크 발급 · 수당 ₩18,700/건 |
| **3단계** | 파트너 | 직접 판매 3건 이상 달성 | 강사 인증 과정 신청 자격 |
| **4단계** | 🎓 공인 강사 | 인증 과정 이수 + 실습 원데이클래스 1회 운영 | 강사 Kit · 강의 수입 100% 본인 귀속 |

### 6-3. 원데이클래스 표준 커리큘럼 (3시간)

## 7. V3 핵심 기능 — 무엇을 자동화하는가  

V3의 핵심은 **"소스 파일만 업로드하면 AI가 대본 → 영상 → 업로드를 자동 완주한다"** 는 것입니다.

### 6-1. 전체 자동화 파이프라인

```
[사용자] 소스 파일 업로드 (기사, 참고자료, 아이디어 메모)
                ↓
[STEP 1]  세계관 생성
          Gemini가 장르·배경·갈등 구조·6챕터 스토리아크 자동 설계
          사용자 확정: 갈등 해소 방식 + 서술 시점 (2가지만 선택)
                ↓
[STEP 2]  캐스팅
          27명 고정 인물 풀에서 트롭 태그 매칭으로 결정론적 선발
          TTS 성우 자동 배정 (character JSON voice_id 기반)
                ↓
[STEP 3]  대본 생성 (6챕터, 1일 1챕터)
          PRE-PASS: Save the Cat 7비트 청사진 생성
          STEP 1: Gemini로 서사 대본 생성 (2,500~4,000자/챕터)
          STEP 1.5: 3-Judge 교차 검증 (문체·구조·인과 — 이기종 AI 3개)
          STEP 2: 씬/컷 JSON 구조화 (20~30 컷/챕터)
                ↓
[STEP 4]  키프레임 이미지 생성
          캐릭터 JSON + 화풍 JSON + 씬 힌트 조합
          Gemini-free 프롬프트 조합 (API 비용 Zero, 응답 < 50ms)
          fal.ai/flux-lora로 캐릭터 맞춤 이미지 생성
                ↓
[STEP 5]  영상 조립 + 업로드
          씬·컷 단위 MP4 생성 → CapCut 최종 합체
          YouTube + 네이버 시리즈 이중 게재
```

### 6-2. 대본 품질 보증: 3-Judge 교차 검증

> **Gemini가 생성한 대본을 Gemini가 채점하면 같은 편향을 공유한다.**  
> 외부 이기종 모델 3개가 각기 다른 축을 병렬 채점하여 자기 검증 맹점을 해소합니다.

| Judge | 담당 축 | 주력 모델 |
|-------|--------|---------|
| Judge A (Style) | 한국어 문체 원칙 | Cerebras llama-3.3-70b |
| Judge B (Structure) | 비트 구조·대사·클리프행어 | Cerebras gpt-oss-120b |
| Judge C (Causality) | 인과·복선·징후 | NVIDIA Nemotron-70b |

모든 모델은 다중 폴백 체인 + 서킷 브레이커로 운영 — 키 미설정 시 Gemini 단독 폴백 자동 전환.

### 6-3. 세계관 설계 원칙 — AI 평균화 방지

Gemini는 학습 데이터의 통계적 평균을 출력하려는 경향이 있어, 사전 제약 없이 생성하면 "평균적인 로맨스물"만 만들어집니다.

**해법**: 사용자가 2가지 핵심 설정을 사전 확정 → Gemini가 그 틀 안에서 창작

| 설정 항목 | 선택지 수 | 예시 |
|---------|--------|------|
| 갈등 해소 방식 | 12개 (현실적 8 + 비현실적 4) | 법정 폭로 / 내부 고발 / 시간 역행 |
| 서술 시점 | 4개 | 1인칭 주인공 / 1인칭 빌런 / 3인칭 전지적 |

나머지 4개 차원 (배경·관계·사회적 균열·갈등 구조) = Gemini 자유 생성 → 다양성 확보

### 6-4. 자동화 단계별 로드맵 (게이트 점진 제거)

현재는 사용자 개입 게이트를 최대로 열어 품질을 검증하고, 결과물이 표준화되면 게이트를 하나씩 닫습니다.

| 단계 | 개입 게이트 수 | 설명 |
|------|-------------|------|
| **Phase A (현재)** | 5개 | 소스 업로드 / 세계관 / 대본 / 키프레임 / 업로드 — 모두 사용자 확인 |
| **Phase B** | 3개 | 세계관·키프레임 autoApprove ON |
| **Phase C** | 1개 | 대본 autoApprove ON — 소스 업로드만 남음 |
| **Phase D (목표)** | 0개 | 스케줄러 자동화 — 완전 Zero-Hands |

### 7-3. 핵심 설계 원칙

- **Zero-API 원칙**: 외부 유료 API 호출 전, 로컬 로직 또는 무료 API를 우선 활용
- **Pydantic 강제 검증**: 모든 백엔드 입출력을 Pydantic 모델로 검증 (Go 수준 엄격함)
- **프론트 JSON 우선**: 정적 콘텐츠(프롬프트·디자인·가이드)는 `apps/web/src/data/` JSON으로 관리
- **DB 연동 최소화**: Supabase는 사용자 정보·매출·에이전트 상태 등 동적 데이터에만 사용
- **서킷 브레이커**: 3-Judge 외부 모델 3회 연속 실패 시 10분 비활성화 → 자동 복구

### 11-4. MVP 런칭 후 — 주 1회 원데이클래스 서비스 운영 로드맵

> **원칙**: MVP 런칭 직후부터 구매자를 대상으로 **매주 1회 정기 원데이클래스**를 운영한다.  
> 초기에는 운영자(이감독)가 직접 강의하며 콘텐츠를 표준화하고, 이것이 강사 양성 커리큘럼의 원본이 된다.

#### 운영 일정 (주간 사이클)

| 요일 | 채널 | 대상 | 내용 |
|------|------|------|------|
| **매주 토요일 오전 10시** | 온라인 (Zoom/Google Meet) | 당주 구매자 전원 | 정기 원데이클래스 — 3시간 과정 |
| **매월 마지막 토요일** | 오프라인 (서울 거점 공간) | 온라인 수료자 중 신청자 | 실습 심화 + 네트워킹 |

#### 온라인 원데이클래스 3시간 표준 진행표

```
[1부 — 60분]  AI 시대 시니어 부업의 현실
  10:00  오프닝: 강사 본인의 ₩59,000 → 강사 과정 스토리
  10:10  시장 데이터로 보는 "지금이 타이밍인 이유"
  10:30  실패 유형 3가지 vs 성공 패턴 1가지
  10:50  Q&A

[2부 — 90분]  V3 툴 실시간 실습
  11:00  V3 접속 → 주제 입력 → 세계관 설정 라이브 시연
  11:20  수강생 각자 스마트폰/PC로 직접 실습 (강사 개별 지원)
  11:50  완성된 웹소설·영상 수강생 발표 (1~2명)
  12:10  "내가 만든 첫 AI 콘텐츠" 인증 스크린샷 공유

  12:20  리셀러 파트너 등록 방법 실습
  12:35  강사 양성 과정 소개 및 다음 단계 안내
  12:50  마무리 & 개별 상담 안내
```

|------|---|--------------|---------|

### 11-5. 강사 양성 체계 구축 로드맵

> **목표**: 운영자가 직접 강의하는 구조에서 벗어나,  
> 인증 강사가 전국에서 자율적으로 원데이클래스를 운영하는 **강사 네트워크**로 확장한다.

#### 강사 양성 4단계 타임라인

```
[ 0기 — 파일럿 ]  MVP 런칭 후 1~8주 (2026 Q2 후반)
  ├── 운영자 직강으로 표준 커리큘럼 완성
  ├── 강의 전 과정 녹화 → 강사 교육용 영상 아카이브 구축
  └── 수강생 피드백 반영 → 커리큘럼 v1.0 확정

[ 1기 — 강사 인증 파일럿 ]  2026 Q3 (Phase B)
  ├── 0기 수료자 중 강사 지원자 모집 (목표: 10명)
  ├── 4주 과정: 커리큘럼 숙지 → 모의 강의 → 실습 원데이클래스 1회
  ├── 인증 심사: 수강생 만족도 4.0 이상 + 콘텐츠 완성률 80% 이상
  └── 1기 공인 강사 배출 → 자율 클래스 운영 시작

[ 2기 — 강사 네트워크 확장 ]  2026 Q4 (Phase C)
  ├── 1기 강사 추천 + 공개 모집 병행 (목표: 30명)
  ├── 지역별 강사 배치: 서울·경기·부산·대구·광주 거점 확보
  ├── 강사 전용 슬랙 채널 + 월 1회 강사 정기 미팅
  └── 강사가 강사를 양성하는 "강사 멘토링" 구조 도입

[ 3기 이후 — 자율 확산 ]  2027~ (Phase D)
  ├── 강사 50명 이상 → 운영자 개입 없이 클래스 자생적 운영
  ├── 온라인 강의 플랫폼(클래스101·탈잉 등) 입점 강사 지원
  └── 강사 수익 추적 대시보드 → 우수 강사 인센티브 제도 운영
```

#### 강사 인증 기준

| 항목 | 기준 | 측정 방법 |
|------|------|---------|
| 커리큘럼 숙지도 | 모의 강의 평가 통과 | 운영자·선배 강사 2인 평가 |
| 실습 완성도 | V3로 콘텐츠 3편 이상 직접 제작 | 결과물 제출 |
| 강의 만족도 | 수강생 평점 4.0 이상 (5점 만점) | 강의 후 설문 |
| 수익화 경험 | 리셀러 직접 판매 3건 이상 | 플랫폼 내 판매 기록 |

#### 강사 활성화 후 운영 구조

```
[운영자]
    │ 커리큘럼 업데이트 · 강사 교육 · 품질 관리
    ▼
[공인 강사 네트워크]
    ├── 온라인 강사 A — 매주 토요일 Zoom 강의 (수강생 10~20명)
    ├── 온라인 강사 B — 매주 일요일 Zoom 강의
    ├── 오프라인 강사 C — 서울 월 2회
    ├── 오프라인 강사 D — 부산 월 1회
    └── ...
    ▼
[수강생] → 새로운 ₩59,000 소비자 → 다시 강사 경로로
```

#### 원데이클래스 운영 KPI

| 지표 | Phase A 목표 | Phase B 목표 | Phase C 목표 |
|------|-----------|-----------|-----------|
| 주간 온라인 클래스 수강생 | 10명/회 | 30명/회 | 50명/회 × 2회 |
| 누적 수료자 | 50명 | 500명 | 2,000명 |
| 공인 강사 수 | 0명 (파일럿) | 10명 (1기) | 30명 (2기) |
| 강사 채널 신규 ₩59,000 판매 | — | 월 30건 | 월 150건 |
